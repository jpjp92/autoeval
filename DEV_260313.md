## 📅 향후 워크플로우 고도화 계획 (Refined Plan)

사용자가 문서의 내부 계층 구조를 미리 알기 어렵다는 점을 반영하여, 다음과 같이 프로세스를 고도화할 계획입니다.

### Phase 1: Data Ingestion (Vectorization)
- **기능**: 파일 업로드 시 텍스트 추출 및 벡터화하여 `doc_chunks`에 우선 저장.
- **변경 사항**: 이제 업로드 시 계층 구조를 직접 입력하지 않음 (단순 인제스션 집중).
- **UI**: 업로드 및 정규화 완료 후, 첫 몇 개의 **청크 프리뷰**를 보여주어 정상 처리 확인 기능 추가.

### Phase 2: Hierarchy Construction (Granular Labeling)
- **핵심 문제**: 문서 전체에 동일한 계층을 부여할 경우 데이터의 다양성 및 검색 정확도 저하.
- **개선 로직 (Refined 4-Step Strategy)**:
    1. **Master Schema Discovery**: 문서 전체 맥락에서 유효한 L1 후보군 도출.
    2. **Sample Granular Tagging (NEW)**: 임의의 3~5개 청크에 대해 계층(L1-L2-L3)을 미리 생성하여 사용자에게 검증용 프리뷰 제공.
    3. **Optimal Mapping**: 사용자가 샘플의 품질을 확인한 후, 전체 청크에 대해 일괄 비동기 태깅 실행.
- **결과**: 사용자는 AI가 각 청크를 어떻게 분류하는지 눈으로 직접 확인하고 전체 업데이트를 승인할 수 있음.
- **UI**: L1 후보 선택 후 '샘플 분석 보기'를 통해 청크별 [본문 - L1-L2-L3] 매칭 사례를 리스트로 표시.

### Phase 3: Data Generation & Evaluation
- **기능**: Phase 2에서 확정된 실제 문서 계층 트리를 기반으로 QA 생성 대상을 선택하여 고도로 정제된 데이터셋 구축.
