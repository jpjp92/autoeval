### 2-1. Phase 1 — Backend API 엔드포인트

**신규**: `GET /api/dashboard/metrics`

```python
# 반환 구조
{
  "summary": {
    "total_qa": 76,                # qa_gen_results 합산
    "avg_final_score": 0.92,       # qa_eval_results.final_score 평균
    "total_documents": 3,          # doc_chunks distinct filename
    "pass_rate": 98.5              # valid_qa / total_qa * 100
  },
  "recent_jobs": [
    {
      "job_id": "gen_20260318_...",
      "type": "generation",
      "source_doc": "프롬프트엔지니어링.pdf",
      "model": "claude-sonnet",
      "total_qa": 28,
      "created_at": "2026-03-18T23:38:31Z"
    },
    ...
  ],
  "grade_distribution": {
    "A+": 2, "A": 1, "B+": 1, "B": 0, "C": 0, "F": 0
  },
  "score_trend": [
    {"date": "2026-03-18", "score": 0.92, "doc": "테스트데이터_1.pdf"},
    {"date": "2026-03-18", "score": 0.89, "doc": "프롬프트엔지니어링.pdf"},
    ...
  ]
}
```

**데이터 소스:**

| 항목               | SQL                                                    | 테이블              |
| ------------------ | ------------------------------------------------------ | ------------------- |
| total_qa           | `SUM(jsonb_array_length(각 doc의 qa_list))`          | `qa_gen_results`  |
| avg_final_score    | `AVG(final_score)`                                   | `qa_eval_results` |
| total_documents    | `COUNT(DISTINCT source_doc)`                         | `qa_gen_results`  |
| pass_rate          | `AVG(valid_qa::float / total_qa) * 100`              | `qa_eval_results` |
| recent_jobs        | `ORDER BY created_at DESC LIMIT 10`                  | 양쪽 UNION          |
| grade_distribution | `GROUP BY final_grade`                               | `qa_eval_results` |
| score_trend        | `SELECT final_score, created_at ORDER BY created_at` | `qa_eval_results` |

**구현 위치**: `backend/api/dashboard_api.py` 신규 또는 `generation_api.py`에 추가

## 5. UI 스타일 리뉴얼 계획 — Glassmorphism Light

### 5-0. 스타일 방향 결정

| 후보                       | 장점                      | 이 프로젝트 적합성                   |
| -------------------------- | ------------------------- | ------------------------------------ |
| **글래스모피즘**     | 모던, 세련, 레이어 깊이감 | ✅ 데이터 카드에 미세 적용 시 효과적 |
| **네오브루탈리즘**   | 개성, 명확한 경계         | ❌ 데이터 집약 평가 플랫폼에 부적합  |
| **완전 Flat (현재)** | 심플                      | 밋밋함, 카드 간 구분 약함            |

**결정: Glassmorphism Light + Flat 혼합**

- 데이터 테이블/차트: flat 유지 (가독성 우선)
- 카드/사이드바: 미세 glass 효과 (세련됨)
- 배경: gradient mesh (밋밋함 해소)

---

### 5-1. 전체 배경 개선

```
현재: bg-slate-50 (단색)

개선: gradient mesh 배경
  bg-gradient-to-br from-slate-50 via-indigo-50/30 to-slate-100
  + 고정 gradient blob (선택)
    <div class="fixed inset-0 -z-10">
      <div class="absolute top-0 right-0 w-96 h-96 bg-indigo-100/40 rounded-full blur-3xl" />
      <div class="absolute bottom-0 left-0 w-96 h-96 bg-sky-100/30 rounded-full blur-3xl" />
    </div>
```

**변경 파일**: `App.tsx` — 메인 `<div>` 배경 클래스

## 9. 청크 콘텐츠 후처리 계획 (Content Post-Processor)

### 9-0. 문제 정의

현재 청킹 후 content 예시:

```
컨텍스트

- 유네스코(UNESCO)는 ...
[표] | 설명 | | 예시 | | --- | --- | --- | | Primary | 인공지능 시스템의 개발... | 특정 알고리즘... |
출처 : UNESCO(2023). "Ethical impact assessment..."
```

| 문제           | 내용                                                  |
| -------------- | ----------------------------------------------------- |
| 인라인 테이블  | `\| col \| col \|` 가 한 줄로 압축 → 테이블 구조 소실 |
| 출처 혼재      | `출처 :` 가 본문 마지막에 섞여 QA 생성 시 노이즈    |
| 구조 정보 소실 | 표/목록/본문 구분 없이 단순 텍스트 저장               |
| 가독성 저하    | LLM이 표 구조를 파악하기 어려워 QA 품질 저하 가능     |

---

### 9-1. 후처리 파이프라인 위치

```
PDF 파싱 → 청킹 → normalize_text()
                        ↓
              [신규] content_postprocessor()   ← 여기 추가
                        ↓
              Embedding → Supabase 저장
```

---

### 9-2. 구현 모듈

**신규 파일**: `backend/preprocessing/content_postprocessor.py`

#### 주요 함수

```python
def postprocess_chunk(content: str) -> PostprocessResult:
    """
    청크 콘텐츠 후처리 진입점
    Returns: content(정제된 본문), source(출처), content_type, has_table
    """
```

| 함수                              | 역할                                                 |
| --------------------------------- | ---------------------------------------------------- |
| `extract_source_citation(text)` | `출처 :` / `Source:` 패턴 감지 → 본문/출처 분리 |
| `detect_inline_table(text)`     | `\| .* \| .* \|` 패턴으로 인라인 테이블 여부 판단     |
| `reconstruct_table(text)`       | 인라인 표 → 멀티라인 Markdown 표로 재구성           |
| `normalize_list_items(text)`    | 불릿/번호 목록 정규화 (`-`, `①②③` 등)         |
| `classify_content_type(text)`   | `table` / `list` / `text` / `mixed` 분류     |

---

### 9-3. 세부 로직

#### 출처 분리

```python
# 감지 패턴
CITATION_PATTERNS = [
    r'출처\s*[:：].*$',
    r'자료\s*[:：].*$',
    r'Source\s*[:：].*$',
    r'\*\s*출처.*$',
]
# → metadata["source_citation"] 에 저장, 본문에서 제거
```

#### 인라인 테이블 재구성

```python
# Before (현재 상태)
"[표] | 설명 | | 예시 | | --- | --- | --- | | Primary | 내용A | 예시A | | Secondary | 내용B | 예시B |"

# After (후처리 결과)
"""
| 설명 | 예시 |
| --- | --- |
| Primary | 내용A | 예시A |
| Secondary | 내용B | 예시B |
"""
```

#### content_type 분류 → metadata 저장

```python
{
  "content_type": "mixed",   # text / table / list / mixed
  "has_table": true,
  "has_list": true,
  "source_citation": "UNESCO(2023). Ethical impact assessment...",
}
```

## 15. 백엔드 서버 기동 오류 수정 & Swagger 그룹화 ✅ 완료 (2026-03-19)

### 15-1. `No module named 'backend'` 오류 수정

**원인**: `config/supabase_client.py` (re-export wrapper)에서 `from backend.db.xxx import` 사용
→ `uv run main.py`를 `backend/` 안에서 실행하면 `main.py` L18이 `backend/` 자체를 sys.path에 추가하므로 `backend` 패키지를 찾지 못함

**수정**: `config/supabase_client.py` 내 7개 import 경로

| Before | After |
| ------ | ----- |
| `from backend.db.base_client import ...` | `from db.base_client import ...` |
| `from backend.db.qa_generation_repo import ...` | `from db.qa_generation_repo import ...` |
| (나머지 5개 동일 패턴) | `from db.xxx import ...` |
