## 2. Dashboard 실데이터 연동 계획

### 2-0. 현재 문제점

Dashboard 전체가 **하드코딩 더미 데이터**로 구성됨.

| 컴포넌트 | 현재 | 문제 |
|----------|------|------|
| `StatsCards.tsx` | Total QA: 14,205 / Score: 0.86 / Cost: $42.50 / Failed: 142 | 전부 하드코딩 |
| `DashboardOverview.tsx` | 최근 작업 RUN-0039~42 | 하드코딩 배열 |
| `ActivityChart.tsx` | Token Usage Mon~Sun | 하드코딩 배열 |
| API Usage 카드 | GPT-5.1: 4.5M/10M | 하드코딩 + 결제 기능 없음 |

---

### 2-1. Phase 1 — Backend API 엔드포인트

**신규**: `GET /api/dashboard/metrics`

```python
# 반환 구조
{
  "summary": {
    "total_qa": 76,                # qa_gen_results 합산
    "avg_final_score": 0.92,       # qa_eval_results.final_score 평균
    "total_documents": 3,          # doc_chunks distinct filename
    "pass_rate": 98.5              # valid_qa / total_qa * 100
  },
  "recent_jobs": [
    {
      "job_id": "gen_20260318_...",
      "type": "generation",
      "source_doc": "프롬프트엔지니어링.pdf",
      "model": "claude-sonnet",
      "total_qa": 28,
      "created_at": "2026-03-18T23:38:31Z"
    },
    ...
  ],
  "grade_distribution": {
    "A+": 2, "A": 1, "B+": 1, "B": 0, "C": 0, "F": 0
  },
  "score_trend": [
    {"date": "2026-03-18", "score": 0.92, "doc": "테스트데이터_1.pdf"},
    {"date": "2026-03-18", "score": 0.89, "doc": "프롬프트엔지니어링.pdf"},
    ...
  ]
}
```

**데이터 소스:**

| 항목 | SQL | 테이블 |
|------|-----|--------|
| total_qa | `SUM(jsonb_array_length(각 doc의 qa_list))` | `qa_gen_results` |
| avg_final_score | `AVG(final_score)` | `qa_eval_results` |
| total_documents | `COUNT(DISTINCT source_doc)` | `qa_gen_results` |
| pass_rate | `AVG(valid_qa::float / total_qa) * 100` | `qa_eval_results` |
| recent_jobs | `ORDER BY created_at DESC LIMIT 10` | 양쪽 UNION |
| grade_distribution | `GROUP BY final_grade` | `qa_eval_results` |
| score_trend | `SELECT final_score, created_at ORDER BY created_at` | `qa_eval_results` |

**구현 위치**: `backend/api/dashboard_api.py` 신규 또는 `generation_api.py`에 추가

### 2-2. Phase 2 — StatsCards 실데이터 교체

| 현재 (더미) | 개선 (실데이터) | 데이터 소스 |
|------------|---------------|------------|
| Total QA Generated: 14,205 | **Total QA**: Supabase 합산 | `summary.total_qa` |
| Avg Quality Score: 0.86 | **Avg Final Score**: 실제 평균 | `summary.avg_final_score` |
| Total API Cost: $42.50 | **Total Documents**: 업로드 문서 수 | `summary.total_documents` |
| Failed Validations: 142 | **Pass Rate**: Quality 통과율 % | `summary.pass_rate` |

> API Cost는 정확한 추적 기능 없으므로 → 문서 수로 대체

**변경**: `StatsCards.tsx`
- 하드코딩 배열 → `useEffect` + `fetch('/api/dashboard/metrics')`
- 로딩 스켈레톤 추가

### 2-3. Phase 3 — 최근 파이프라인 실행 실데이터

**현재**: 하드코딩 4건 (RUN-0039~42)
**개선**: `recent_jobs` API 연동 → 최근 10건

| 컬럼 | 데이터 소스 |
|------|------------|
| 작업 ID | `job_id` (gen_... / eval_...) |
| 유형 | `type` (생성 / 평가) |
| 대상 | `source_doc` |
| 모델 | `model` |
| 상태 | 현재는 전부 completed (jobs 테이블 연동 후 다양화) |
| 항목 수 | `total_qa` |
| 시간 | `created_at` → relative time (n분 전 / 어제) |

**변경**: `DashboardOverview.tsx`
- 하드코딩 `recentRuns` 배열 → API 호출 결과로 교체

### 2-4. Phase 4 — 차트 실데이터 교체

#### 차트 1: 평가 점수 추이 (현재 Token Usage 대체)

```
X축: 날짜 (또는 평가 순번)
Y축: final_score (0~1)
데이터: score_trend[]
차트 유형: AreaChart 유지 (현재 recharts 이미 사용 중)
```

#### 차트 2: 등급 분포 (API Usage 카드 대체)

```
A+ / A / B+ / B / C / F 분포
데이터: grade_distribution
차트 유형: 수평 BarChart 또는 도넛 차트
```

**변경**: `ActivityChart.tsx` → 이름 변경 `ScoreTrendChart.tsx`
