## 📋 예정 계획

### P1 — 청크 콘텐츠 후처리 (Content Post-Processor)

**문제**: 현재 content 예시

```
[표] | 설명 | | 예시 | | --- | --- | --- | | Primary | 내용A | ...
출처 : UNESCO(2023). "Ethical impact assessment..."
```

| 문제 | 내용 |
| ---- | ---- |
| 인라인 테이블 | `\| col \|` 가 한 줄로 압축 → 테이블 구조 소실 |
| 출처 혼재 | `출처 :` 가 본문에 섞여 QA 생성 시 노이즈 |
| 구조 정보 소실 | 표/목록/본문 구분 없이 단순 텍스트 저장 |

**구현 위치**: `backend/preprocessing/content_postprocessor.py` (신규)

**파이프라인 위치**:
```
PDF 파싱 → 청킹 → normalize_text() → [신규] content_postprocessor() → Embedding → Supabase
```

**주요 함수**:

| 함수 | 역할 |
| ---- | ---- |
| `extract_source_citation(text)` | `출처 :` / `Source:` 패턴 → 본문/출처 분리, metadata 저장 |
| `detect_inline_table(text)` | `\| .* \| .* \|` 패턴 감지 |
| `reconstruct_table(text)` | 인라인 표 → 멀티라인 Markdown 표 재구성 |
| `normalize_list_items(text)` | 불릿/번호 목록 정규화 |
| `classify_content_type(text)` | `table` / `list` / `text` / `mixed` 분류 → metadata 저장 |

**구현 우선순위**:

| 순서 | 작업 | 난이도 | 효과 |
| ---- | ---- | ------ | ---- |
| 1 | `extract_source_citation()` | 낮음 | 즉각적 노이즈 제거 |
| 2 | `detect_inline_table()` + `reconstruct_table()` | 중 | LLM 파악률 향상 |
| 3 | `normalize_list_items()` | 낮음 | 목록 가독성 개선 |
| 4 | `classify_content_type()` + metadata 저장 | 중 | QA 생성 프롬프트 분기 기반 |
| 5 | QA 생성 프롬프트에서 content_type 활용 | 중 | QA 품질 직접 향상 |

### P2 — Docker 구성 (로컬 개발 전용, 아카이브)

> 배포는 Render(Backend) + Vercel(Frontend) 사용 — Docker는 로컬 환경 통일 목적

**목표 구조**:
```
autoeval/
├── docker-compose.yml          # 프로덕션
├── docker-compose.dev.yml      # 개발 오버라이드 (볼륨 마운트, HMR)
├── backend/Dockerfile          # FastAPI + uv
└── frontend/Dockerfile         # Vite 빌드 → Nginx
```

**backend/Dockerfile**:
```dockerfile
FROM python:3.12-slim
COPY --from=ghcr.io/astral-sh/uv:latest /uv /usr/local/bin/uv
WORKDIR /app
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen --no-dev
COPY backend/ ./backend/
ENV PYTHONPATH=/app
EXPOSE 8000
CMD ["uv", "run", "uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**frontend/Dockerfile** (멀티스테이지):
```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY frontend/package*.json ./
RUN npm ci
COPY frontend/ ./
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY frontend/nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

**nginx.conf**:
```nginx
server {
    listen 80;
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
    location /api/ {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**구현 순서**:

| 순서 | 작업 |
| ---- | ---- |
| 1 | `backend/Dockerfile` 작성 |
| 2 | `frontend/nginx.conf` 작성 |
| 3 | `frontend/Dockerfile` 멀티스테이지 |
| 4 | `docker-compose.yml` 프로덕션 (healthcheck 포함) |
| 5 | `docker-compose.dev.yml` 개발 오버라이드 |
| 6 | `.env.example` 커밋 |
