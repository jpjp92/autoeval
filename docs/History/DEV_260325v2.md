## 1. Settings — 탭 네비게이션 개선

**파일**: `frontend/src/components/settings/SettingsPanel.tsx`

### 1-1. 현재 구조 문제

```
┌────────────┬─────────────────────────────────────┐
│  Profile   │                                     │
│  API Keys  │      Content Area                   │
│  Pipeline  │      (flex-1)                       │
│  Notif...  │                                     │
│  (w-64)    │                                     │
└────────────┴─────────────────────────────────────┘
```

| 문제 | 영향 |
|------|------|
| 좌측 사이드바 `w-64 (256px)` 고정 | 콘텐츠 가로 공간 낭비 |
| 항목이 4개뿐인데 세로 나열 | 레이아웃 무게감 과함 |
| Pipeline 탭에서 React Flow 가로 축소 | 도식도 가독성 저하 |

### 1-2. 개선안 — 상단 탭 방식

```
┌──────────────────────────────────────────────────────┐
│  👤 Profile  │  🔑 API Keys  │  ⬡ Pipeline  │  🔔   │  ← 탭 바 (~48px)
├──────────────────────────────────────────────────────┤
│                                                       │
│              Content Area (full width)                │
│                                                       │
└──────────────────────────────────────────────────────┘
```

**탭 스타일**: underline 방식

```tsx
// active
"border-b-2 border-indigo-500 text-indigo-600"

// inactive
"border-b-2 border-transparent text-slate-500 hover:text-slate-800 hover:border-slate-300"
```

**레이아웃 변경 요약**

| 항목 | 현재 | 변경 |
|------|------|------|
| 외부 flex 방향 | `flex-row` | `flex-col` |
| 좌측 nav div | `w-64` 사이드바 | 제거 |
| 탭 바 | 없음 | 상단 `border-b` 언더라인 탭 추가 |
| 콘텐츠 영역 | `flex-1 p-8` | 동일 (full width) |
| Pipeline 높이 | `calc(100% - 88px)` | `calc(100% - 48px)` (탭 바만 제외) |

**변경 범위**: `SettingsPanel.tsx` 단일 파일

## 2. Notification — 알림 시스템 설계

**파일 (예정)**: `frontend/src/components/layout/Header.tsx` + `frontend/src/lib/notificationStore.ts`

### 2-1. 알림이 필요한 이유

QA 생성·평가는 비동기 Job으로 처리되어 수 분~수십 분 소요.
사용자가 다른 탭(Dashboard, Evaluation 등)에서 작업 중에도 완료·실패를 즉시 인지할 수 있어야 함.

---

### 2-2. 알림 유형 및 발생 시점

#### ① Job 완료 / 실패 — 가장 핵심

| 이벤트 | 유형 | 내용 예시 |
|--------|------|----------|
| QA 생성 완료 | 성공 | "테스트데이터_5.pdf — QA 42개 생성 완료" |
| QA 생성 실패 | 오류 | "QA 생성 실패: Claude API 응답 오류 (job_id: abc123)" |
| 평가 완료 | 성공 | "평가 완료 — 평균 점수 0.82 / 등급 A" |
| 평가 실패 | 오류 | "평가 실패: 타임아웃 (42/100 완료 후 중단)" |

#### ② 인제스션 단계

| 이벤트 | 유형 | 내용 예시 |
|--------|------|----------|
| 업로드·임베딩 완료 | 성공 | "테스트데이터_5.pdf — 청크 87개 저장 완료" |
| 업로드 실패 | 오류 | "파일 파싱 오류: 손상된 PDF" |
| 계층 분석 완료 | 정보 | "H1 4개 / H2 18개 / H3 42개 마스터 생성 완료" |
| 청크 태깅 완료 | 정보 | "87개 청크 태깅 완료 (__admin__ 3개 제외)" |

#### ③ 품질 경고 — 평가 결과 이상 감지

| 이벤트 | 유형 | 내용 예시 |
|--------|------|----------|
| 낮은 평균 점수 | 경고 | "평균 점수 0.48 — 기준(0.65) 미달, 생성 설정 검토 권장" |
| 모순 감지 | 경고 | "groundedness ↔ completeness 모순 QA 3건 감지" |
| F등급 과다 | 경고 | "F등급 비율 32% — 프롬프트 또는 청크 품질 점검 필요" |

#### ④ 시스템 상태

| 이벤트 | 유형 | 내용 예시 |
|--------|------|----------|
| API Rate Limit 임박 | 경고 | "Claude API 사용량 80% 도달 — 생성 속도 조절 중" |
| DB 연결 오류 | 오류 | "Supabase 연결 실패 — 재시도 중" |

| 레벨 | 색상 | 아이콘 | 용도 |

### 2-4. UI 구조

#### 헤더 벨 아이콘 + 배지

```
[🔔 3]  ← 읽지 않은 알림 수 배지 (red dot 또는 숫자)
```

- 배지: 읽지 않은 알림 수, 최대 `9+` 표기
- 클릭 시 드롭다운 패널 오픈 (click-outside 닫기)

#### 드롭다운 패널

```
┌─────────────────────────────────────┐
│  알림  ·  3개 미확인    [모두 읽음]  │
├─────────────────────────────────────┤
│ ✓  QA 생성 완료          2분 전     │
│    테스트데이터_5.pdf · 42개        │
├─────────────────────────────────────┤
│ ⚠  낮은 평균 점수        5분 전     │
│    0.48 — 기준 미달                 │
├─────────────────────────────────────┤
│ ✕  평가 실패             12분 전    │
│    타임아웃 (42/100)                │
└─────────────────────────────────────┘
```

- 최대 20개 표시, 이후 스크롤
- 읽지 않은 항목: 좌측 `border-l-2` 강조 + 배경 연하게
- 개별 클릭 시 해당 Job 결과 페이지로 이동 (job_id 기반)
- "모두 읽음" 버튼으로 일괄 처리

### 2-5. 상태 관리 설계

**저장소**: `frontend/src/lib/notificationStore.ts` (Zustand 또는 Context)

```typescript
interface Notification {
  id: string;
  level: 'success' | 'info' | 'warning' | 'error';
  title: string;
  description?: string;
  timestamp: Date;
  read: boolean;
  jobId?: string;       // 클릭 시 이동 대상
  targetTab?: string;   // 'generation' | 'evaluation' | 'dashboard'
}
```

**발생 시점**: 각 API 폴링 완료·실패 콜백에서 `addNotification()` 호출

```typescript
// 예: generation status polling 완료 시
if (status === 'completed') {
  addNotification({
    level: 'success',
    title: 'QA 생성 완료',
    description: `${filename} · ${qaCount}개`,
    jobId: job_id,
    targetTab: 'generation',
  });
}
```

**영속성**: `localStorage` 저장 (새로고침 후에도 유지, 최대 50개)

### P1/P2 상세 설계 — 상세 뷰 통합 표시

#### 현재 구조 (2-section)

```
┌──────────────────────────┐  ┌──────────────────────────┐
│  RAG Triad        0.812  │  │  품질 평가        0.795  │
│  관련성    [reason]       │  │  사실성    [reason]       │
│  근거성    [reason]       │  │  완전성    [reason]       │
│  명확성    [reason]       │  │  구체성    [reason]       │
│                          │  │  간결성    [reason]       │
└──────────────────────────┘  └──────────────────────────┘
```

#### 개선 구조 (1-section 통합)

```
┌──────────────────────────────────────────────────────────┐
│  품질 평가         RAG 0.812  /  품질 0.795              │
├──────────────────────────────────────────────────────────┤
│  관련성     [reason]     ← RAG 차원                      │
│  근거성     [reason]     ← RAG 차원                      │
│  명확성     [reason]     ← RAG 차원                      │
│  완전성     [reason]     ← Quality 차원                  │
│  ─────────────────── (구형 데이터 추가 표시) ────────── │
│  사실성     [reason]     ← 구형 Quality 차원             │
│  구체성     [reason]     ← 구형 Quality 차원             │
│  간결성     [reason]     ← 구형 Quality 차원             │
└──────────────────────────────────────────────────────────┘
```

#### 하위 호환 판별 기준

```typescript
// reason 값 존재 여부로 신·구 자동 판별 (필드는 모두 optional로 유지)
const isLegacy = !!(qa.factuality_reason || qa.specificity_reason || qa.conciseness_reason);

// 신규: ragDimensions(3) + [완전성] → 4개
// 구형: ragDimensions(3) + [사실성, 완전성, 구체성, 간결성] → 7개
```

#### 점수 표시 방식

- 헤더에 `RAG 0.812 / 품질 0.795` 형태로 두 평균값 병기
- 두 점수 중 낮은 쪽 색상 기준으로 배경 결정 (기존 로직 유지)

#### HTML Report 동일 적용

- `grid-template-columns: 1fr 1fr` → 단일 컬럼 전환
- ragRows + qualRows를 단일 `detail-section-qual` 블록으로 병합
- 헤더에 두 점수 병기 (`RAG 0.812 · 품질 0.795`)
