> 프로덕션 배포 환경 트러블슈팅 — Supabase pause 복구 + 파일 업로드 5MB 초과 에러 수정

## 1. 세션 배경

Vercel(프론트엔드) + Render(백엔드) 배포 후 웹에서 서버가 동작하지 않는 문제 확인.  
원인 분석 및 파일 업로드 제한 이슈까지 연속 대응.

---

## 2. 문제 1 — Supabase DB pause

**증상:** 웹에서 모든 API 요청 실패, 서버 자체가 동작하지 않는 것처럼 보임  
**원인:** Supabase 무료 플랜은 7일 비활성 시 DB 자동 pause  
**해결:** Supabase 대시보드 → "Restore project" 클릭 (1~3분 소요)

**재발 방지 권고:**
- Render 슬립 + Supabase pause 동시 방지: GitHub Actions 스케줄러로 `/health` 매일 ping

```yaml
# .github/workflows/keepalive.yml
name: Keep Alive
on:
  schedule:
    - cron: '0 12 * * *'
jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - run: curl -f https://autoeval-uccr.onrender.com/health
```

## 3. 문제 2 — 파일 업로드 FUNCTION_PAYLOAD_TOO_LARGE

**증상:** 5MB / 200페이지 PDF 업로드 시 `Request Entity Too Large` 에러  
**원인:** Vercel 프록시(rewrite)를 통해 업로드 시 **4.5MB 페이로드 제한**에 걸림

```
Browser → Vercel rewrite (/api/ingestion/upload) → Render   ← Vercel이 4.5MB로 차단
```

**해결:** 파일 업로드만 Vercel 프록시를 우회하여 Render에 직접 전송

### 코드 변경 (`frontend/src/lib/api.ts`)

```ts
// 추가
const UPLOAD_BASE = import.meta.env.VITE_UPLOAD_BASE_URL ?? '';

// uploadDocument() 수정
export async function uploadDocument(formData: FormData): Promise<IngestionStartResponse> {
  return apiFetch<IngestionStartResponse>(`${UPLOAD_BASE}/api/ingestion/upload`, { method: 'POST', body: formData });
}
```

- `VITE_UPLOAD_BASE_URL` 미설정 시 빈 문자열 → 로컬에서는 기존 상대경로로 동작
- 나머지 API 호출은 기존 Vercel 프록시 유지

### 환경변수 설정

| 플랫폼 | 변수명 | 값 |
|--------|--------|----|
| Vercel | `VITE_UPLOAD_BASE_URL` | `https://autoeval-uccr.onrender.com` |
| Render | `CORS_ORIGINS` | `https://autoeval-v1.vercel.app` |

> **주의:** `VITE_*` 변수는 빌드 타임에 번들에 포함됨 → Vercel 환경변수 추가 후 배포해야 적용됨

### 업로드 제한 현황 (수정 후)

| 구간 | 제한 |
|------|------|
| Vercel 프록시 | 4.5MB ← 우회 |
| Render nginx (`client_max_body_size`) | 200MB |
| FastAPI 백엔드 | 제한 없음 |

## 4. 배포 아키텍처 요약

```
Browser
  ├── 일반 API 요청 (/api/...) → Vercel rewrite → Render FastAPI
  └── 파일 업로드 (/api/ingestion/upload) → Render FastAPI (직접)

Render FastAPI → Supabase (pgvector)
```
