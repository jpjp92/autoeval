#### [이슈 A] 밑줄 표시 기반 정보 생성 → **완전 배제**

**문제**: 컨텍스트에 "밑줄은 역대 최대 수출액을 나타낸다"는 기준 설명은 있지만, PDF → 텍스트 변환 과정에서 실제 밑줄 표시 자체는 소실됨. 생성 모델이 이 기준을 보고 특정 수치가 "역대 최대"라고 단정 → 컨텍스트에서 확인 불가능한 주장 생성.

**판단**: 이런 케이스는 생성 단계에서 막아야 함. 평가 기준을 완화해서 허용할 성격의 문제가 아님.

**개선 방향**: `plan_before_generate` Step 2에 "표현 형식(밑줄, 볼드, 색상 등)에 의존하는 정보는 텍스트 변환 후 소실되므로 매핑 불가로 처리" 추가.

#### [이슈 B] 배율 계산 및 비율 비교 규칙 누락 → **`evaluate_all_with_reasons` 프롬프트 강화**

**문제**: `evaluate_groundedness`에는 "Calculated values (difference, sum, ratio) from two context-present numbers = Strong" 규칙이 있지만, **실제 평가 파이프라인에서 사용하는 `evaluate_all_with_reasons`의 groundedness 기준에는 이 규칙이 누락**되어 있음.

- qa_index 38: 2001년 109백만달러, 2023년 952백만달러 → 8.7배 계산. 두 수치 모두 컨텍스트에 존재 → Strong 처리해야 하나 None으로 판정.
- qa_index 48: 2014년 35백만달러, 2016년 75백만달러 → 2배 증가. 두 수치 모두 컨텍스트에 존재 → 동일 문제.

**`evaluate_all_with_reasons` 현행 groundedness 기준**:
```
supported=true ONLY IF: direct quote OR unambiguous paraphrase
supported=false IF: requires inference, interpretation, deduction
```
→ 계산형 파생정보 규칙이 아예 없음. 배율/비율 계산이 "deduction"으로 분류되어 None 판정.

**개선 방향**: `evaluate_all_with_reasons` groundedness 기준에 계산형 파생정보 예외 규칙 추가:
```
EXCEPTION — Derived calculations: If the answer states a calculated value
(difference, sum, ratio, multiple, percentage) and BOTH source numbers exist
explicitly in the context, treat this claim as supported=true (equivalent to Strong).
e.g., context has "2001년 109백만달러" and "2023년 952백만달러"
→ answer stating "약 8.7배 증가" is FULLY GROUNDED.
```

#### [이슈 C] 추이 해석 표현 처리 기준 결정

**문제**: 수치 기반으로 도출되는 추이 해석 표현("2년 연속 감소세", "반등 성공", "절반이 넘는 국가")을 현행 기준이 추론(None/Weak)으로 처리.

**케이스 분류**:

| 표현 | 컨텍스트 근거 | 판단 |
|---|---|---|
| "2년 연속 감소세" (2021 -1.3%, 2022 -6.2%) | 두 감소 수치 모두 컨텍스트에 존재 | 허용 범위 |
| "반등 성공" (2023 +66.2%) | 수치 컨텍스트에 존재 | 허용 범위 |
| "절반이 넘는 국가" (67/129) | 두 수치 모두 컨텍스트에 존재 | 허용 범위 |
| "상당하다는 평가" | 컨텍스트에 없는 주관적 평가 | 금지 |
| "핵심 시장" | 컨텍스트에 없는 레이블링 | 금지 |

**결정 방향**: 수치 기반 논리적 귀결 표현은 **Medium(0.7-0.8)** 으로 허용. 단 조건:
1. 근거 수치 **2개 이상**이 컨텍스트에 명시적으로 존재
2. 해석이 **단방향 수치 비교로 직접 도출** 가능 (증감 방향만 판단, 가치 판단 없음)
3. 주관적 레이블("상당히", "핵심", "성공적") 없이 사실 기술 형태

**`evaluate_all_with_reasons` 추가 기준**:
```
MEDIUM-LEVEL EXCEPTION — Trend expressions derived from context numbers:
If the answer uses trend language ("consecutive decline", "rebound") AND the underlying
numbers that support this conclusion are ALL explicitly in the context, treat as Medium
(supported=true for scoring purposes), NOT as inference.
Condition: factual direction only — no subjective labels ("significant", "impressive", "key market").
```

## 6. 복수 지표 누락 및 조건형 전제 누락 원인 분석

### 6.1 공통 패턴 — RAG relevance vs Quality completeness 불일치

두 케이스(qa_index 4, 18) 모두 **RAG relevance=1.0이지만 Quality completeness가 낮음**.

- RAG `relevance`: 질문의 주제/의도에 부합하는지 평가 → 답변이 "성장 지표에 대해 말하고 있다"면 1.0
- Quality `completeness`: 컨텍스트 vs 답변을 직접 비교해 누락 수치/조건 감지 → 더 엄격

즉, 생성 단계에서 불완전한 답변이 만들어졌고 RAG relevance가 이를 통과시키는 구조.

### 6.2 [케이스 A] 복수 지표 답변 일부 누락 (qa_index 4)

**증상**: "장기 성장 지표" 질문에서 일부 지표만 나열하고 종결.
누락된 항목: 수출액(9.52억달러), 증가율(24.4%), 2024년 1월 수치(39.4%), 10년 연속 기대치.

**근본 원인**: 현행 `[답변 완전성]` 규칙이 키워드("모두", "항목별로") 기반 트리거.
"장기 성장 지표들"처럼 **복수형 명사**로 묻는 질문은 트리거가 없음.
또한 plan 단계에서 **"답변에 포함해야 할 항목 목록"을 사전에 만드는 단계가 없어** 생성 중 누락이 발생.

**개선 방향**:
- `plan_before_generate`에 "질문이 확정되면, 답변에 포함할 컨텍스트 항목을 먼저 열거" 단계 추가
- `[답변 완전성]` 규칙에 "복수형 명사(지표, 항목, 수치, 조건 등)가 질문에 있으면 컨텍스트에서 해당 주제의 수치를 전수 열거" 명시

### 6.3 [케이스 B] 조건형 질문 전제 서술 누락 (qa_index 18)

**증상**: "지구 몇 바퀴 계산의 전제 조건" 질문에서 총 면발 길이·지구 둘레는 제시했으나 "한 봉지당 50m" 누락.

**근본 원인**: `plan_before_generate`의 답변 검증이 "질문 요소 → 컨텍스트 매핑"에만 집중.
condition/how 타입 질문은 **전제 조건 전체**가 답변에 포함되어야 하는데, 이를 강제하는 규칙이 없음.

**개선 방향**:
- `[답변 완전성]`에 condition/how 타입 전용 규칙 추가: "조건형·방법형 답변은 전제 조건과 분기 항목 전체를 열거"
- `plan_before_generate`에 답변 항목 사전 목록화 단계 연계

### 6.4 개선 방향 — `plan_before_generate` Step 구조 변경

현재 plan 5단계는 모두 **질문 검증**에 집중되어 있음. 답변 완전성 보장을 위한 사전 목록화 단계 추가 필요.

**추가할 Step 6 (안)**:
```
6. 확정된 각 질문에 대해, 답변에 반드시 포함해야 할 컨텍스트 항목을 먼저 열거하세요.
   - 수치 지표가 복수이면 해당 주제의 모든 수치를 목록화
   - condition/how 질문이면 전제 조건과 분기 항목 전체를 목록화
   - 이 목록 전체가 답변에 포함되었는지 확인 후 최종 출력
```

## 11. 결정 사항 및 적용 내역

### 11.1 Groundedness threshold 0.8로 조정

- **이유**: 수치 기반 주관적 표현(정도 부사)은 "틀린 답변"이 아닌 "표현 방식" 수준.
  groundedness 0.8대를 미달로 처리하면 멀쩡한 QA가 지속 탈락.
- **적용 위치**: `pipeline.py` `_distribution_stats` groundedness 호출부에 threshold=0.8 명시.
- **메모**: 향후 생성 모델이 정도 부사를 줄이면 threshold를 다시 높일 수 있음.

### 11.2 Step 1 목록화 범위 확장 — 서술적 배경 정보 포함

- **이유**: purpose/condition형 질문에서 정성적 배경(배경 설명, 사회적 맥락, 문화적 요인 등)이
  Step 1 목록화에서 누락 → 답변에도 누락.
- **방향**: 특정 문서(K-문화)에 종속되지 않는 범용 표현으로 "서술적 배경 정보(배경·맥락·이유·효과 설명 등)"를 Step 1 목록화 대상에 추가.
- **적용 위치**: `plan_before_generate` Step 1 (USER_TEMPLATE_KO_V1 + build_user_template).
