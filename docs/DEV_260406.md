## 오늘 적용한 변경 (`backend/generators/prompts.py`)

### P3 — purpose 생성 조건 강화 (Hallucination 방지)

**적용 위치 1**: `_CORE_PRINCIPLES_KO` `<intent_types>` (line ~335)
- 기존: "명시되었거나 효과로부터 유추 가능한 경우" ← **유추 허용 상태였음**
- 변경: 명시적 목적 문구 조건(`~을 목적으로`, `~을 위하여`, `~하는 것을 목적`, `~이유는`, `~배경은`) 있을 때만 허용
- 추가: "결과·효과·조문 내용에서 목적을 추론하는 것 엄격히 금지" 명시

**적용 위치 2**: `build_user_template()` `<groundedness_check>` (P3 블록 신규 추가)
- 동일 조건을 유저 프롬프트 체크 단계에서 재확인

**적용 위치 3**: `_generate_intent_examples()` purpose 예시 라인
- "이유 명시된 경우만" → 명시적 목적 문구 조건 + 추론 금지로 구체화

**기대 효과**: results8에서 발생한 qa#79 (0점), qa#76, #78, #91 패턴 방지
- 법조문에 목적 미명시 시 purpose 생성 자체를 차단

### P4 — Faithfulness 강화 (추론성 표현 금지)

**적용 위치 1**: `_CORE_PRINCIPLES_KO` `<constraints>` — `[추론 경계 강화]` 블록 신규 추가
- 컨텍스트 외 절차·순서·인과관계·수식어 추론 금지
- 구체 금지 예시: "선행하고", "이를 근거로 병행하여", "원활하고 전문적인", "안건으로 상정하여"
- 두 사실이 각각 존재해도 관계 미명시 시 관계 서술 금지

**적용 위치 2**: `build_user_template()` `<groundedness_check>` — `[추론성 표현 금지]` 블록 신규 추가

**기대 효과**: results8 qa#78 ("선행하고", "병행하여"), qa#91 ("원활하고 전문적인"), qa#80 ("안건으로 상정하여") 패턴 방지

### P5 — List 완전성 강화 (누락 방지)

**적용 위치 1**: `_CORE_PRINCIPLES_KO` `<constraints>` — `[답변 완전성]` 블록 신규 추가
- '각 호', '다음과 같다', '다음 각 호', '①②③', '가.나.다.' 열거 표현 감지 시 전 항목 포함 의무화
- 복수 항목 암시 질문에 컨텍스트 내 모든 항목 포함 의무화
- N개 항목 중 M < N개만 포함하는 불완전 답변 금지

**적용 위치 2**: `build_user_template()` `<groundedness_check>` — `[list 완전성]` 블록 신규 추가

**기대 효과**: results8 qa#40 (5가지 중 1가지), qa#54 (내부 직무만), qa#82 (일부 누락), qa#130 (5가지 중 2가지) 패턴 방지

## 검토 필요 항목

### ⚠️ P2 유저 프롬프트 미반영

`_CORE_PRINCIPLES_KO` `<constraints>`의 `[comparison형 답변 — 변화 방향 서술 필수]` 규칙이
`build_user_template()` `<groundedness_check>`에 없음.
- 시스템 프롬프트로만 제어 중 → 유저 프롬프트 체크 단계에 추가 검토
- **우선순위**: P6 재검증 테스트 시 함께 처리

### ⚠️ P6 — P2 효과 재검증 미실시

results8은 법률 문서(수치 비교 없음)라 P2(comparison 변화 방향) 효과 확인 불가.
- **필요 작업**: 통계/수치 문서(라면 수출 통계 등)로 P1+P2+P3 적용 후 재생성
- 기대 확인 포인트: comparison 답변에 "2023년 대비 증가/감소" 등 방향 서술 포함 여부

### 검증 필요 — P3/P4/P5 효과 확인 (다음 results)

results8 동일 문서(개인정보 보호법) 재생성 권장:
- P3 목표: purpose 타입 생성 건수 17건 → 대폭 감소 + quality 실패 qa#79/#76/#78/#91 패턴 제거
- P4 목표: groundedness/faithfulness 감점 사례 감소
- P5 목표: list 타입 답변 완전성 향상 (qa#40/#54/#82/#130 유형 제거)

### 검토 — `SYSTEM_PROMPT_KO_V1` / `USER_TEMPLATE_KO_V1` 처리 방침

현재 두 정적 프롬프트는 코드에 선언만 돼 있고 미사용.
- 오늘 적용된 P3/P4/P5 변경 내용이 반영되어 있지 않음
- deprecated 주석 처리 또는 동적 빌더와 동기화 중 선택 필요
