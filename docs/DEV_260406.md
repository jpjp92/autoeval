# DEV_260406 — 프롬프트 P3/P4/P5 적용 및 일관성 정비

## 배경

DEV_260405에서 results8 분석 후 도출된 미적용 항목(P3/P4/P5)을 오늘 적용.

### 프롬프트 호출 경로

```
worker.py
  └─ build_system_prompt()  ← _CORE_PRINCIPLES_KO 포함
  └─ build_user_template()  ← 동적 생성
       └─ _generate_qa(system_prompt=..., user_template=...)  # 주입

qa_generator.py  _generate_qa()
  └─ system_prompt가 None이면 SYSTEM_PROMPT_KO_V1 fallback 사용
  └─ user_template이 None이면 USER_TEMPLATE_KO_V1 fallback 사용
```

- **정상 경로** (`worker.py`): 동적 빌더(`build_system_prompt` + `build_user_template`)가 주입되어 작동
- **Fallback 경로** (`qa_generator.py` 직접 호출 또는 주입 누락 시): `SYSTEM_PROMPT_KO_V1` / `USER_TEMPLATE_KO_V1` 사용
- **`_CORE_PRINCIPLES_KO`**: `build_system_prompt()` 내부 코어 블록 (정상 경로 시스템 프롬프트의 실체)

→ `SYSTEM_PROMPT_KO_V1` / `USER_TEMPLATE_KO_V1`은 미사용이 아닌 **fallback 프롬프트**로, 오늘 적용된 P3/P4/P5가 반영되지 않은 상태임

## BUG-3 — 통계 문서 H3 전부 제외 (chars < 300 필터)

### 증상
- 통계 문서(승용차 교역 현황) 업로드 후 QA Pipeline 카테고리 새로고침 시 "전체"만 표시
- 로그: `Hierarchy filtered: 14 h3 node(s) excluded (chunk count < 1 or chars < 300)`
- 생성 로그: `h1=None, h2=None, h3=None` (전체 범위 강제)

### 원인
`backend/db/hierarchy_repo.py:108`
```python
if filter_for_qa and (cnt < MIN_CHUNKS_FOR_QA or chars < MIN_CONTENT_CHARS):
```
- `MIN_CHUNKS_FOR_QA = 1` → `cnt < 1` 조건은 사실상 불가능 (노드 존재 시 최소 1청크)
- **실제 트리거**: `chars < MIN_CONTENT_CHARS (300)` — 통계 문서 H3 청크가 표 데이터 위주(100~200자)로 300자 미달 → 14개 H3 전부 제외 → h1_list 비워짐

### 수정 방안 검토

| 방안 | 내용 | 리스크 |
|------|------|------|
| A. `MIN_CONTENT_CHARS` 하향 (300 → 100) | 단순, 즉시 적용 가능 | 빈 노드 노출 가능성 |
| B. OR → AND 변경 | `cnt < MIN AND chars < MIN` | 두 조건 모두 실패할 때만 제외 — 더 보수적 | 낮음 |
| C. `MIN_CONTENT_CHARS`를 도메인별 동적화 | 근본 해결 | 구현 복잡 |

**권장**: 방안 B — `or` → `and` 변경. 청크가 1개 이상 존재하면 chars 조건과 무관하게 노출. 실질적으로 `cnt < 1`(존재 불가)과 `chars < 300` 모두 만족해야 제외되므로 사실상 chars 필터 해제와 동일하나 의도는 명확.

## P7 — 비대칭 비교 질문 방지 (comparison 대칭 검증)

### 근거 케이스 — results13 qa#19

**질문**: "2,000cc 초과 대형 차량의 수입에서 휘발유 차량과 경유 차량의 감소율 및 전분기 대비 추이는 어떻게 나타납니까?"

**컨텍스트 데이터 비대칭**:

| 대상 | 전년 동기 대비 감소율 | 전분기 대비 추이 |
|------|:--:|:--:|
| 휘발유 차량 | 50.0% ✅ | **없음** ❌ |
| 경유 차량 | 91.6% ✅ | △86.3% → 91.6% ✅ |

**실패 구조**: 경유 차량에만 전분기 추이 데이터가 있는데, 두 차량 유형 모두에 전분기 추이를 묻는 질문 생성 → 휘발유 차량 전분기 추이 답변 불가 → context_relevance 0.5 / completeness 0.6

**평가 결과**: 관련성 0.6 / 근거성 1.0 / 맥락성 0.5 / 완전성 0.6 → avg 0.735
- 근거성 1.0: 답변 내용은 컨텍스트에 근거 (답변 자체의 문제가 아님)
- 맥락성 0.5: 컨텍스트가 질문을 커버 못 함 → **질문 생성 단계 오류**

**`<plan_before_generate>` 2단계 통과 실패 원인**: 경유 차량의 전분기 추이가 컨텍스트에 있으니 두 대상 모두 묻는 질문 생성. 휘발유 차량의 전분기 추이 매핑 여부를 **대칭 검증**하지 않음.

### P7 규칙 정의

> comparison 타입 질문에서 두 대상(A, B)에 동일 속성을 묻는 경우, **양쪽 모두** 해당 속성이 컨텍스트에 있는지 확인.
> 한쪽에만 데이터가 있는 경우:
> - 해당 속성을 질문에서 제거하거나
> - 데이터가 있는 대상만으로 질문 범위 축소
