# 2026-03-30: QA 품질 평가 및 통계 분석 고도화 산출물

이 문서는 QA 품질 평가 엔진의 `완전성(Completeness)` 지표 개선 사항과 `데이터셋 통계(Layer 1-B)`의 산출 방법론 및 지표 정의를 상세히 정리합니다.

---

## 1. 품질 평가: 완전성 (QA Quality: Completeness)

LLM 답변이 질문의 모든 요구사항을 충실히 반영했는지 측정하는 지표입니다.

### 1.1 측정 원리: 질문 분해 (Decomposition)
- **원리**: 복합적인 질문을 원자 단위의 서브 질문(Atomic sub-questions)으로 분해하여 평가의 객관성을 확보함.
- **예시**: "A와 B의 차이점은 무엇이며, 각각의 장점은?"
  - 서브 질문 1: A와 B의 차이점이 설명되었는가?
  - 서브 질문 2: A의 장점이 언급되었는가?
  - 서브 질문 3: B의 장점이 언급되었는가?

### 1.2 채점 기준 및 커버리지 (Coverage)
- **커버리지 계산**: `답변된 서브 질문 수 / 전체 서브 질문 수` 비율로 산출.
- **점수 산출 (Scaling)**: 
  - 커버리지 100%: 10점 (완벽)
  - 커버리지 80% 이상: 8-9점 (사소한 누락)
  - 커버리지 50% 미만: 5점 이하 (핵심 내용 누락)
- **누락 요소 탐지**: 답변에서 다루지 못한 측면을 `missing_aspects`로 추출하여 사용자에게 구체적인 피드백 제공.

## 2. 데이터셋 통계: Layer 1-B 산출 방법론

데이터셋 자체의 건전성과 다양성을 물리적으로 측정하는 지표입니다.

### 2.1 다양성 (Diversity)
*   **정의**: 데이터셋이 얼마나 다양한 주제와 어휘를 포함하고 있는가.
*   **산출 방식**:
    - **인텐트 엔트로피 (60%)**: Shannon Entropy($-\sum p_i \log_2 p_i$)를 통해 인텐트 분포의 균형도를 측정. (균등할수록 고득점)
    - **어휘 다양도 (TTR, 40%)**: 고유 토큰 수 대비 전체 토큰 수 비율($Unique / Total$). (중복 단어가 적을수록 고득점)
*   **목표**: 특정 유형의 질문에만 매몰되지 않은 균형 잡힌 데이터셋 확보.

### 2.2 중복도 (Duplication)
*   **정의**: 서로 다른 질문 쌍 간의 텍스트 유사성 정도.
*   **산출 방식**: `SequenceMatcher` 알고리즘을 사용해 질문 텍스트 간 유사도가 70% 이상인 쌍(Near-duplicate)을 탐지.
*   **감점 로직**: `10 - (중복률 / 2)`. 중복률이 20%를 넘어가면 0점 처리하여 데이터 중복을 엄격히 관리.

### 2.3 편향도 (Skewness)
*   **정의**: 특정 소스 문서(docId)에 질문이 과도하게 집중되어 있는 정도.
*   **산출 방식**: 전체 데이터셋 중 단일 문서가 차지하는 최대 비중(`doc_max_ratio`)을 기준으로 구간 채점.
*   **기준**: 
    - 30% 이하: 10점 (이상적 분포)
    - 50% 이하: 8점
    - 80% 이하: 5점
    - 80% 초과: 2점

### 2.4 충족성 (Sufficiency)
*   **정의**: 데이터 분석 및 학습에 필요한 필수 정보가 물리적으로 갖춰졌는가.
*   **산출 방식**:
    - **필드 채움률**: 질문, 답변, 컨텍스트, 문서ID, 인텐트 등 5개 필수 필드의 기입 여부 전수 조사.
    - **물리적 볼륨**: 질문/답변의 평균 길이를 통해 텍스트가 분석하기에 충분한 정보량을 담고 있는지 보조적으로 판단.

## 4. 평가 상태 판정 로직 개선 (2026-03-30)

### 4.1 개요
사례 #4와 같이 점수는 높지만(0.7 이상) 실제로는 '근거 오류' 등의 결함이 감지된 경우, 기존의 단순 점수 기반 '성공' 판정을 지양하고 **'보류(Hold)'**로 분류하여 사용자 검토를 유도하도록 로직을 개선함.

### 4.2 하이브리드 판정 로직 (Hybrid Score-Error Logic)
- **실패 (Fail)**: 품질 및 Triad 평균 점수가 모두 0.7점 미만인 경우.
- **보류 (Hold)**: 
  - 점수가 0.7점 미만인 지표가 하나라도 있는 경우.
  - **예외 처리**: 점수가 모두 0.7점 이상이나, 백엔드 판정 결과 결함(`pass: false` 또는 `failure_types` 존재)이 발견된 경우.
- **성공 (Success)**: 모든 점수가 0.7점 이상이며, 어떠한 결함도 감지되지 않은 경우.

### 4.3 주요 수정 및 수정 내역
- **대시보드 UI (`getQAStatus`)**: 백엔드 전달 데이터(`pass`, `failure_types`)를 최우선으로 고려하는 판정 로직 적용 및 Null Guard 강화.
- **엑셀(XLSX) 내보내기**: 대시보드 UI와 동일한 상태 판정 로직으로 동기화하여 리포트 일관성 확보.
- **HTML 리포트**: 
  - 내장 스크립트(`qaStatus`)를 고도화하여 동일 로직 적용.
  - 구형 브라우저 호환성을 위해 `??` 연산자 제거 및 표준 JavaScript 문법(`indexOf`, `function`) 사용.
  - 루프 내 정의되지 않은 변수 참조(`ReferenceError`) 수정 완료.

## 5. 개별 지표 시각화 및 리포트 고도화 (2026-03-30)

평가 결과의 투명성을 높이기 위해 단순 평균 점수 노출을 넘어 각 지표별 세부 점수와 근거를 통합 시각화하도록 개선함.

### 5.1 QA 상세 보기 UI 개선 (3컬럼 레이아웃)
- **레이아웃 리팩토링**: `[지표명] [점수] [사유]`의 3개 열 구조를 도입하여 정보의 위계 정립.
  - **지표명 (Label)**: 관련성, 근거성, 맥락성, 완전성 등 명시적 레이블링.
  - **점수 (Score)**: 소수점 3자리까지 표시하며 모노스페이스 폰트 적용.
  - **사유 (Reasoning)**: 점수에 대한 LLM의 판단 근거를 우측에 배치하여 가독성 확보.
- **점수대별 가시성(Color Logic)**:
  - `Emerald (≥0.85)`: 우수 (Pass)
  - `Amber (0.70 ~ 0.85)`: 보통 (Attention)
  - `Rose (<0.70)`: 주의 (Fail/Hold)

### 5.2 HTML 내보내기 리포트 동기화
- **오프라인 리포트 고도화**: 대시보드 UI와 동일한 3컬럼 레이아웃을 오프라인 HTML 파일에서도 그대로 구현.
- **데이터 무결성 확보**: 리포트 생성 시 단순 `avg_score` 뿐만 아니라 개별 항목 점수를 전수 매핑하여 전달하도록 `exportUtils.ts` 로직 보강.
- **하위 호환성**: 개별 점수 필드가 없는 구형 데이터 로드 시 점수 열에 `-` 실장하여 안정적인 렌더링 보장.

### 5.3 백엔드 데이터 필드 보강
- **`pipeline.py` / `evaluation_api.py`**:
  - `qa_preview` 배열 및 상세 조회 API(`_build_export_detail`) 응답에 `relevance`, `groundedness`, `context_relevance`, `completeness` 점수 필드를 추가하여 프론트엔드와 데이터 모델 동기화 완료.

## 9. 대시보드 헤더 및 KPI 카드 UI 고도화 (2026-03-30)

리포트의 전문성을 높이고 인터랙션 오류를 해결하기 위해 대시보드 상단 영역을 전면 재설계함.

### 9.1 헤더 레이아웃 재설계 (Report-Centric Design)
- **정보 계층 구조 확립**:
  - **대타이틀**: "QA 성능 평가 리포트"로 명칭을 통일하고 강조함.
  - **메타데이터 태그**: 모델, 데이터셋, 일시 정보를 단순 나열하지 않고 `Model:`, `Dataset:` 등 레이블과 함께 배치하여 직관성 강화.
  - **ID 제거**: 시각적 노이즈가 되는 해시태그 형태의 ID 번호를 제거하여 깔끔한 타이틀 라인 유지.
- **버튼 스타일 통일**: History 및 Export 버튼을 상단 태그들과 동일한 **알약 형태(Pill-shape, rounded-full)**로 변경하여 시각적 일관성 확보.

### 9.2 KPI 카드 인터랙션 및 레이어링 수정
- **툴팁 트리거 정교화 (Named Group)**:
  - **문제**: KPI 카드 전체(Parent)에 호버 시 내부의 'i' 아이콘 툴팁이 함께 나타나는 간섭 발생.
  - **해결**: Tailwind의 `group/info` 명명된 그룹 기능을 사용하여, 마우스가 정확히 **'i' 아이콘에 올라갔을 때만** 설명창이 나타나도록 트리거 영역을 한정함.
- **Z-Index 계층 역전 해결**:
  - **문제**: 상단 드롭다운 메뉴(History, Export)가 하단 KPI 카드 컨텐츠 뒤로 숨겨지는 현상.
  - **해결**: 헤더 컨테이너에 `z-40`, KPI 카드 영역에 `z-20`을 차등 부여하여 드롭다운이 항상 최상단에 노출되도록 보장.
- **툴팁 가시성**: 툴팁이 하단 차트 요소들에 가려지지 않도록 KPI 카드 호버 시 개별 카드의 우선순위를 `z-30`으로 동적 상향 조정.

## 10. 의도 분포 트리맵 시각화 및 UX 고도화 (2026-03-30)

기존의 단순 차트를 대체하여 질문 의도 분포를 한눈에 파악할 수 있는 계층형 트리맵(Treemap) 시각화를 도입하고 가독성을 대폭 개선함.

### 10.1 반응형 트리맵 (Responsive Treemap) 도입
- **시각적 비중 표현**: 질문 수량에 비례한 면적 분할 방식으로 데이터셋의 의도 편향성을 직관적으로 감지 가능.
- **면적 기반 폰트 티어링 (5단계)**:
  - 셀의 면적($Width \times Height$)을 계산하여 5단계(초소형~최대)로 분류.
  - **초소형 셀 (비중 5% 이하)**: 8~9px의 초소형 폰트를 적용하고 레이블을 숨겨 시각적 겹침 현상 방지.
  - **대형 셀**: 최대 18px의 굵은 폰트를 사용하여 핵심 지표 강조.
- **텍스트 가독성 최적화**:
  - `paintOrder="stroke"` 속성과 반투명 검은색 스트로크를 결합하여 어떤 배경색 위에서도 글자가 뚜렷하게 보이도록 개선.
  - `drop-shadow`를 추가하여 텍스트의 입체감과 배경 분리도 향상.

### 10.2 인터랙션 및 애니메이션 정돈
- **진입 애니메이션 변경**: 아래에서 위로 올라오는 과도한 슬라이드 효과를 제거하고, 부드럽게 커지며 나타나는 `fade-in zoom-in-95` 조합으로 변경하여 세련된 첫인상 제공.
- **호버 피드백 고도화**: 마우스 호버 시 해당 셀에 `brightness`와 `saturate` 효과를 부여하고 섀도우를 깊게 하여 선택된 영역을 명확히 강조.

### 10.3 오프라인 리포트(HTML) 및 UI 편의성 개선
- **HTML 내보내기 고도하기**: 대시보드와 동일한 급의 질문 의도 분포 트리맵 시각화를 `exportToHTML`에 실장함. 
  - **무의존 재귀 알고리즘**: 별도의 외부 라이브러리 없이 순수 SVG로 동작하는 Slice-and-Dice 기반 트리맵 알고리즘 구현.
  - **가변 폰트 임계값 최적화**: 좁거나 작은 셀(너비 18px 이상)에서도 수치(%)가 잘리지 않고 노출되도록 4단계(Large/Medium/Small/Micro) 폰트 스케일링 적용.
- **의도 분포(Intent) 하위 호환성 강화**:
  - 기존 6종 스킴 외에 구형 데이터(`summary`, `confirmation` 등) 8종 이상을 모두 수용할 수 있도록 `INTENT_KR` 및 컬러 매핑을 14종 규모로 확장.
### 10.4 UI/UX 디테일 및 레이아웃 최적화
- **상세 평가 테이블 가시성 개선**: 
  - 테이블 헤더에 `whitespace-nowrap`을 적용하여 "Triad 점수" 등이 두 줄로 넘어가는 현상 해결.
  - 지표 컬럼(품질, Triad, 실패유형) 너비를 `w-20`에서 `w-24`로 확장하여 정렬 아이콘과의 간섭 배제.
- **전체 레이아웃 공백 최적화**: 
  - `App.tsx`의 메인 컨테이너 상단 패딩을 `pt-2`에서 `pt-6`으로 상향하여 스티키 헤더(브레드크럼)와의 시각적 답답함 해소.
- **워크스페이스 정리**: 작업 과정에서 실수로 생성된 빈 파일(`frontend/=`, `frontend/22`)을 삭제하여 환경 무결성 유지.

## 11. 대시보드 시각적 고도화 및 프리미엄 디자인 (2026-03-30)

대시보드의 전문성을 극대화하고 데이터 시각화의 감성 품질을 높이기 위해 핵심 UI 컴포넌트들을 'Premium & Minimal' 컨셉으로 전면 리뉴얼함.

### 11.1 데이터 통계 게이지 애니메이션 보정
- **문제**: 점수가 10.0(만점)일 때 애니메이션 없이 이미 채워진 상태로 노출되어 시각적 역동성이 떨어짐.
- **해결**: `isReady` 상태와 `strokeDashoffset` 초기값을 동기화하여, 10.0 점수일 때도 0에서부터 시계 방향으로 부드럽게 차오르는 애니메이션 실장.
- **기술 사양**: 초기 `offset`을 전체 둘레(`circ`)로 설정 후, 컴포넌트 마운트 시 목표치까지 트랜지션 처리.

### 11.2 통합 품질 점수 차트 (Premium Card Layer)
기존의 단순 리스트 형태를 입체적인 카드 레이어 구조로 전환하여 데이터의 위계와 몰입감을 강화함.
- **디자인 컨셉**: **Frosted Glass Card**. `bg-slate-50/50` 배경과 미세한 보더(`border-slate-100`)를 사용하여 입체감 부여.
- **프로그레스 바 고도화**:
  - **그라데이션**: `Emerald-Teal` (성공), `Amber-Orange` (주의), `Rose-Red` (실패) 3단 레이아웃.
  - **네온 글로우**: 각 상태색에 맞춘 부드러운 글로우 섀도우(`shadow-[0_0_12px]`) 적용.
  - **인터랙티브 팁**: 프로그레스 바 끝단에 화이트 글로우 포인트 추가 및 호버 시 확대 효과(Pulse Point).
- **레이아웃**: RAG/품질 구분 태그를 소형 뱃지 형태로 상단 배치하여 텍스트 가독성 확보.

### 11.3 액션 버튼 스타일 리파인
'History' 및 'Export' 버튼의 디자인을 대시보드의 신뢰감 있는 룩앤필에 맞춰 조정함.
- **형태 변경**: 기존 원형(`rounded-full`)에서 신뢰감 있는 **직사각형(`rounded-xl`)**으로 변경.
- **인터랙션**: 호버 시 `shadow-md` 강화 및 `-translate-y-0.5` 리프트 효과를 추가하여 클릭감을 유도.
- **일관성**: 버튼의 워딩과 기본 크기는 유지하여 기존 사용자 경험을 보존.

### 11.4 HTML Export 리포트 시각적 정돈
- **통계 레이아웃**: 중앙 통계 섹션의 '통합 점수' 배치를 하단 지표들과 조화롭도록 미세 조정.
- **정렬 기본값**: 내보내기 시 데이터 정렬의 기본값을 `ID (1번부터)` 순으로 설정하여 리포트의 체계성 확보.

### 11.5 히스토리 전환 시 애니메이션 일관성 확보 (2026-03-31)
- **문제**: 히스토리 전환 시 이전 점수와 새 점수가 동일(예: 10.0 -> 10.0)할 경우 React가 변화를 감지하지 못해 애니메이션 리셋이 생략되는 현상 발생.
- **해결**: `MetricRadialGauge`의 `useEffect` 의존성을 `stat.A`에서 `stat` 객체 전체로 확장하여, 값이 같더라도 데이터의 맥락이 바뀌면 무조건 애니메이션을 리셋(`setIsReady(false)`)하도록 수정.
- **결과**: 모든 히스토리 전환 상황에서 게이지가 0부터 차오르는 역동적인 연출을 일관되게 유지함.

### 11.6 마이너 UI 워딩 및 가시성 개선 (2026-03-31)
- **의도 분포 툴팁**: 설명 항목 중 '6종'이라는 표현을 '분류'로 변경하여 데이터 가변성에 유연하게 대응하도록 수정.
- **QA 생성 패널**: 자동 평가 토글 라벨을 '생성 후 4-Layer 자동 실행'에서 'QA 자동 평가 실행'으로 간결하게 변경하여 가독성 향상.
