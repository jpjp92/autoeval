## 📋 9. 정성적 품질 평가 (Quality Evaluation)

**평가 방법:** 자동 휴리스틱 기반 점수 (0~5점)  
**평가 도구:** [quality_eval.py](../../../works/autoeval/data_check/quality_eval.py)  
**평가 기준:**
1. **근거성** (Groundedness): 컨텍스트에만 기반하는가
2. **자연성** (Naturalness): 실제 고객이 물어볼 법한가
3. **의도정확성** (Intent Accuracy): 태깅된 의도가 정확한가
4. **명확성** (Clarity): 질문과 답변이 명확한가
5. **형식** (Format): JSON 완전성 및 필드 정확성

### 9-0. 평가 방법 상세 (휴리스틱 로직)

#### 1️⃣ 형식 (Format) - auto_eval_format()

```
시작 점수: 5점

감점 조건:
- 필수 필드 누락 (q, a, intent, answerable): -1점 (각각)
- intent 타입이 8가지 중 하나가 아님: -1점
  (factoid, numeric, procedure, definition, list, boolean, why, how)
- q 또는 a가 비어있음: -1점

최종: max(0, score)
```

**예시:**
```json
✅ 완벽: 모든 필드 정상 (q, a, intent, answerable 모두 있음, intent="why")
→ 5/5

❌ 부족: intent 필드 누락
→ 4/5

❌ 부족: intent="설명" (잘못된 타입)
→ 3/5
```

---

#### 2️⃣ 근거성 (Groundedness) - auto_eval_groundedness()

```
시작 점수: 5점

감점 조건:
- 답변이 20자 미만: -2점 (근거 부족, 너무 짧음)
- 답변이 300자 초과: -1점 (외부지식 사용 가능성)
- "모르", "확실하지 않", "알 수 없", "정보 없" 포함: -2점

최종: max(0, score)
```

**예시:**
```
Q: "이게 뭔가요?"
A: "잘 모르겠습니다."
→ 5 - 2 = 3/5 (거부 응답, 근거 부족)

Q: "온라인 셀프서비스로 뭘 할 수 있나요?"
A: "온라인 셀프서비스는 로그인 후 계정 관리, 요금 확인, 
    탈퇴 등을 직접 처리할 수 있습니다. (약 120자)"
→ 5/5 (명확한 근거, 적절한 길이)
```

**특징:**
- 텍스트 길이로 판단 (URL, 컨텍스트 미참고)
- 진정한 컨텍스트 검증은 불가능 (외부 정보 필요)

---

#### 3️⃣ 의도정확성 (Intent Accuracy) - auto_eval_intent()

```
시작 점수: 5점

의도별 키워드 검사 (질문 텍스트 기준):
- why:        ["왜", "why", "이유", "reason", "cause"] 포함 여부
- how:        ["어떻게", "how", "방법", "절차", "step"] 포함 여부
- definition: ["무엇", "what", "정의", "definition", "의미"] 포함 여부
- list:       ["어떤", "무엇", "종류", "옵션", "list"] 포함 여부
- boolean:    ["가능", "가능한가", "할수있나"] 포함 여부
- numeric:    답변(a)에 숫자(0-9) 포함 여부

키워드/특성 미포함 시: -1점

최종: max(0, score)
```

**예시:**
```
intent="why", Q="링고 서비스는 뭐야?" (태그는 why지만 질문에 "왜" 없음)
→ 감점 → 4/5 (불일치)

intent="why", Q="왜 미성년자는 요금할인이 안 되나요?" ("왜" 포함)
→ 감점 없음 → 5/5 (일치)

intent="numeric", A="12개입니다." (숫자 포함)
→ 5/5

intent="numeric", A="여러 개입니다." (숫자 없음)
→ 4/5 (numeric 답변이 아님)
```

**한계:**
- 단순 키워드 매칭 (의미 미파악)
- "왜"는 없지만 실제로 why의도일 수 있는 경우 놓침

---

#### 4️⃣ 명확성 (Clarity) - auto_eval_clarity()

```
시작 점수: 5점

감점 조건:
- 질문이 10자 미만: -2점 (너무 짧음, 구체성 부족)
- "그것", "그거", "그런 것", "어떤 것", "뭔가" 포함: -1점 (모호함)
- 답변이 "..." 로 끝나거나 "..."이 2회 이상: -1점 (불완전)

최종: max(0, score)
```

**예시:**
```
Q="그것?" (3자, "그것" 포함)
→ 5 - 2 - 1 = 2/5 (매우 불명확)

Q="VIP 초이스에서 제공하는 제휴사는?" (17자, 명확함)
→ 5/5

A="가능합니다. 단, 같은 달에는..." ("..." 1회)
→ 5 - 1 = 4/5

A="... 그리고 ... 또한 ..." ("..." 3회 이상)
→ 5 - 1 = 4/5
```

---

#### 5️⃣ 자연성 (Naturalness) - auto_eval_naturalness()

```
시작 점수: 5점

감점 조건:
- 질문이 "다." 또는 "가?" 로 끝남: -1점 (기계적)
- 질문이 8자 미만: -2점 (너무 짧음)
- 질문이 100자 초과: -1점 (너무 길고 복잡)
- "나요?", "나요", "인가요?", "나?", "까요?"로 끝나지 않고
  "?" 도 미포함: -1점 (자연스럽지 않은 어체)

최終: max(0, score)
```

**예시:**
```
Q="이게 뭐다." ("다."로 끝남, 4자)
→ 5 - 1 - 2 = 2/5 (부자연스러움)

Q="VIP 초이스는 어떤 혜택을 제공하나요?" (존댓말, 18자)
→ 5/5 (자연스러움)

Q="VIP 초이스가 무엇이고 어떻게 이용하며 언제 사용할 수 있고 
   어디서... (150자 초과, 여러 질문 섞임)"
→ 5 - 1 = 4/5 (너무 길고 복잡)
```

### 9-0(추가). 평가 방법의 한계와 주의

| 기준 | 평가 방식 | 한계 | 개선 방안 |
|------|---------|------|---------|
| **형식** | 필드 존재 여부 | 필드는 있지만 내용 오류 미파악 | LLM 검증 필요 |
| **근거성** | 답변 길이 + 거부 키워드 | 실제 컨텍스트 비교 불가 | DeepEval (Faithfulness) 사용 |
| **의도정확성** | 키워드 매칭 | 의미 이해 불가 | LLM 기반 의도 분류 |
| **명확성** | 길이 + 대명사 + 부호 | 통사적으로 복잡한 문장도 명확할 수 있음 | LLM 문해도 검사 |
| **자연성** | 어체 + 길이 | 문법적으로 자연스러워도 의미상 부자연스러울 수 있음 | LLM 자연스러움 평가 |

**결론:** 현재 휴리스틱 방식은 **빠르고 비용 없음** (계산 기반)이지만, 정확도는 제한적입니다.

**추천 개선 방법:**
1. **1차 필터링**: 휴리스틱 (빠른 선별)
2. **2차 정밀 평가**: DeepEval + LLM (정확도 향상)
   - 예상 비용: ~$0.5~1 (3,318 QA 기준)
