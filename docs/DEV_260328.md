### 문제

- 외부 컨테이너 `bg-white/60 border-t` → 박스 안 박스 중첩으로 어색
- 탭바 `bg-white/80` 추가 레이어로 3중 배경

5. **Pipeline 배경 dot 패턴 제거**: `<Background variant={Dots}>` 컴포넌트 삭제, 라이트/다크 동일 처리

## Phase 5 — 헤더 Notification 시스템

### 기획 배경

AutoEval은 PDF 임베딩, 계층 태깅, QA 생성·평가 등 장시간 비동기 작업이 많아
작업 완료 시점을 사용자에게 명확히 알려줄 필요가 있음.
Settings의 Notifications 탭은 제거하고 헤더 Bell 아이콘으로 통합.

### 알림 발생 이벤트

| 이벤트 | 메시지 예시 | 타입 |
|---|---|---|
| PDF 임베딩 완료 (`onUploadComplete`) | 임베딩 완료 · `doc_v3.pdf` | success |
| 계층 태깅 완료 (`onTaggingComplete`) | 계층 태깅 완료 | success |
| QA 생성 · 평가 시작 (`onEvalComplete`) | QA 생성 완료 · 평가 시작됨 | success |
| 에러 발생 (향후) | 생성 실패 · API 오류 | error |

### 메시지 포맷

```
[타입 아이콘]  제목 (font-medium)          상대 시간
              부제목 (text-xs, 옵션)
```

상대 시간 표기: `방금 전` / `N분 전` / `N시간 전` / `N일 전`
절대 시간은 title 속성(hover)으로 노출.

### 데이터 구조

```ts
interface Notification {
  id: string;          // crypto.randomUUID()
  title: string;
  sub?: string;        // 파일명, 부가 정보
  type: 'success' | 'error' | 'info';
  time: Date;
  read: boolean;
}
```

### 상태 관리 (in-memory, Option A)

```
App.tsx
  notifications: Notification[]
  addNotification(n)  — 신규 알림 prepend
  clearAll()          — 전체 삭제
  markAllRead()       — 드롭다운 열 때 일괄 read = true

  ↓ props
Header.tsx
  Bell 아이콘 + unread 뱃지 (indigo dot)
  클릭 → 드롭다운 (w-80, max-h-72 scroll)
  "전체 삭제" 버튼 (알림 있을 때만)
  빈 상태: Bell 아이콘 + "알림이 없습니다"
```

### 타입별 스타일

| 타입 | 아이콘 | 색상 |
|---|---|---|
| success | `CheckCircle2` | `text-emerald-500 dark:text-emerald-400` |
| error | `XCircle` | `text-red-500 dark:text-red-400` |
| info | `Info` | `text-indigo-500 dark:text-indigo-400` |

### 변경 파일

| 파일 | 작업 |
|---|---|
| `frontend/src/components/layout/Header.tsx` | Bell 아이콘, 드롭다운, `Notification` 타입 export |
| `frontend/src/App.tsx` | notifications 상태, addNotification/clearAll/markAllRead, 콜백 연결 |
| `frontend/src/components/settings/SettingsPanel.tsx` | Notifications 탭 제거 (Bell import 포함) |

### 헤더 버튼 UI 시각적 위계 정리 (260329 연장 작업)

알림(메인)과 다크모드 선택(보조) 간의 기능적 위계를 명확히 하기 위해 헤더 우측 바 UI 시각적 개선 (Option D):
- **버튼 간격 확대 (`gap-2`)**: 오클릭 방지 및 여백 확보
- **알림 기능 강조**: 아이콘 크기(`w-5 h-5`) 및 색상 대비(`slate-600`)를 높여 주 기능임을 명시
- **시각적 분리**: 두 버튼 사이에 흐린 세로 구분선(`|`) 추가하여 성격 분리
- **보조 버튼 위계 하락**: 다크모드 토글 버튼은 기본 투명도 `opacity-80` 및 연한 색상 처리로 시각적 부담 축소 (hover 시 선명해짐)

## Phase 6 — 사이드바 및 대시보드 테이블 UI/UX 고도화 (260329 신규 작업)

### 1. 사이드바 애니메이션 리팩토링 및 레이아웃 안정화
- **로고 및 토글 버튼 이동(Jump) 현상 완벽 제거**: Flex 컨테이너에서의 조건부 컴포넌트 언마운트(`{!isCollapsed}`) 방식을 폐기하고, 텍스트와 로고 블록을 `w-[160px]` ↔ `w-0`으로 넓이 조절(`overflow-hidden`)하여 스무스하게 내용이 잘려나가며(Clipping) 트랜지션 되도록 레이아웃 알고리즘 최적화
- **토글 버튼 고급화**: `absolute` 포지셔닝을 통해 사이드바 컨테이너가 쪼그라들 때 위치 편향 없이 우측 끝(`right-[2px]`)에서 자연스럽게 미끄러지도록 설계했으며, `rounded-full`과 마이크로 인터랙션(`hover:scale-105 active:scale-95`) 적용으로 조작감 고도화
- **메인 메뉴 아이콘 안정화**: 접혀있을 때 내부 여백이 36px로 좁아지는 점을 정확히 계산하여 활성화 배경(Squircle 컨테이너)이 찌그러지거나 아이콘이 흔들리지 않고 무조건 정중앙에 위치하도록 `w-9 h-9 mx-auto` 사이즈로 독립화

### 2. 대시보드 파이프라인 테이블 리디자인
- **Layout Shift 방지**: 데이터 길이에 따라 테이블 높이가 유동적으로 변해 발생하는 UX 불편을 해소하기 위해 `table-fixed` 레이아웃 도입 및 각 컬럼별 고정 너비(비율) 할당 
- **모델 컬러 필(Pills) 도입**: 모델명 텍스트 대신 `inline-block` 컬러 뱃지를 사용하여 모델 다각화 시 시각적 구분력 강화 
- **정렬(Sort) 기능 도입**: 모든 헤더에 오름차순/내림차순 정렬을 추가하고, 중앙 정렬(`text-center`)을 도입하여 UI의 균형감 상승
- **명도 대비 개선**: 라이트 모드에서 행 호버(Hover) 시 하이라이트 배경색 명도 대비가 부족해 잘 보이지 않던 문제를 `bg-indigo-50` 레벨로 교정

### 3. 평가 리포트 차트명 변경
- 대시보드 레이아웃 및 HTML 추출(Export) 리포트 공통 작업
- "RAG Triad + 품질 평가 점수" 네이밍을 디자인/기획 의도에 맞추어 **"통합 품질 평가 점수"**로 일괄 변경

### 4. 대시보드 마이크로 인터랙션(Micro-interactions) 고도화
- **페이지 진입 모션 (Staggered Mount)**: 요약 지표 카드, 파이프라인 테이블, 차트 섹션들이 위에서 아래로 순차적으로 시간차를 두고 등장(`slide-in-from-bottom`)하도록 딜레이 애니메이션 결합 적용
- **Interactive StatsCards**: 상단 요약 지표 카드 호버 시 살짝 떠오르는(`Lift-up`, `-translate-y-1`) 모션, 진해지는 그림자(`shadow-xl`), 아이콘 `scale` 팽창 및 좌측 보더 하이라이트 추가
- **Quick Actions (빠른 실행)**: 호버 시 버튼 텐션(`scale-[1.02]`) 및 클릭 시 압곽(`active:scale-[0.98]`) 반동, 숨어있던 `→` 화살표의 슬라이드 연출 추가
- **차트(ActivityChart) 다이나믹스**: 기본 차트 툴팁을 대시보드 무드에 맞춘 `backdrop-blur-md` 글래스 재질로 변경하고, 호버된 점(Active Dot) 부근에 강렬한 파란색 Glow 광원 효과 적용 시각화

### 5. Evaluation 페이지 뷰 디테일 개선
- **표(Table) 비율/밀도 조정**: 상세 결과 데이터 테이블의 텍스트가 다소 커서 발생하는 여백 뷸균형 개선
- 헤더 영역을 `12px`(`text-xs`)로 축소하고 본문 데이터 텍스트의 베이스라인을 `11px`로 압축, 정렬 화살표도 `9px`로 축소해 정보의 시각적 밀도와 가독성 비율 상승
- 실패 유형 매핑 누락 픽스: 모델이 출력하는 새로운 주요 실패 유형 중 하나인 `poor_context`(문맥 부족)가 바깥 표(Table) 기준의 `FAILURE_CONFIG` 매핑 테이블에 없어 렌더링되지 않던 버그를 해결. `poor_context`를 **"문맥부족"**(bg-sky-50 레이블 뱃지)으로 표기되도록 연동 완료.

## Phase 6.5 — 파일 업로드 고도화 및 네비게이션 개편 (260329 연장)

### 8. Phase 8: Evaluation UI 고도화 & UX 정교화 (작업 완료)
- **Evaluation 헤더 리디자인**:
  - 등급 뱃지(A+, A 등)에 입체적인 섀도우 및 글로우 효과를 적용하여 프리미엄 느낌 부여.
  - 모델명, 평가 일시, 소스 문서 등 메타데이터를 아이콘 기반의 **Pill(캡슐) 스타일**로 그룹화하여 정보 밀도 및 시각적 정돈성 강화.

  - 데이터 부재 시에도 서비스의 완성도를 보여주도록 인디고/퍼플 **그라데이션 글로우 배경**과 플로팅 카드 레이아웃 적용.

- **QA Pipeline 가시성 개선**:
  - 다크모드에서 HTML `select` 옵션의 가시성 문제를 해결하기 위해 불투명 배경(`bg-slate-800`) 및 명시적인 옵션 배경/텍스트 색상 수동 지정.

- **기획 의도**: 단순 시간순 점수 추이보다 "어떤 모델이 가장 우수한가"에 대한 직관적인 데이터 통찰력을 제공하기 위해 기존 그래프를 리더보드 형태의 벤치마킹 보드로 교체.
- **Backend (Aggregation)**:
  - `dashboard_repo.py` 내 크로스 집계 로직 구현. `generation_model`별로 그룹화하여 누적 평균 점수, 합격률(Syntax 통과율), 총 테스트 횟수를 계산.
  - `qa_eval_results`의 메타데이터에서 생성 모델 정보를 직접 추출하도록 최적화.
- **Frontend (UI/UX)**:
  - **1위 강조**: 금메달 아이콘, 골드-인디고 그라데이션 배경, 상단 레인보우 글로우 보더 적용.
  - **브랜드 컬러링**: GPT(기묘한 초록), Claude(오렌지), Gemini(블루) 등 모델 제작사별 고유 컬러를 Pill 및 바 그래프에 매핑.
  - **이중 지표 시각화**: 평균 점수 바(Primary)와 QA 통과 수량 실수치(`32 / 34개`) 및 합격률 뱃지(Secondary)를 하단에 배치하여 정보의 질적/양적 측면을 동시 제공.
- **Motion**: 등급 분포 차트와 동일한 `clipPath` 기반 좌➡️우 reveal 애니메이션을 적용하고, 순위별로 `120ms`씩 순차적 등장 딜레이를 부여하여 리드미컬한 화면 진입 연출.

### 2. 대시보드 비주얼 통일성 및 버그 픽스
- **섹션 아이콘화**: "파이프라인 로그"(`ScrollText`)와 "빠른 실행"(`Rocket`) 헤더에 아이콘을 추가하여 하단 리더보드/분포 차트와 시각적 위계 통일.
- **텍스트 잘림 수정**: 파이프라인 로그 테이블에서 긴 모델명(예: `gemini-3-flash-preview`)이 부자연스럽게 잘리던 문제를 해결. `<td>`의 truncate를 제거하고 내부 칩(`<span>`)에 `max-w`와 `truncate`를 적용해 칩 형태를 유지하며 깔끔하게 잘리도록 교정.

### 3. Evaluation 상세 페이지 UX 정교화
- **버튼 디자인 통일**: 헤더의 "History" 드롭다운과 "Export" 버튼의 너비(`w-36`), 높이(`py-2.5`), 라운딩(`rounded-xl`), 폰트 굵기(`font-bold`)를 완벽히 일치시켜 레이아웃 안정감 확보.
- **텍스트 다이어트**: "Export 리포트"를 "Export"로 축약하여 간결함 극대화.
- **다크모드 명도 교정**: 히스토리 드롭다운에서 현재 선택된 항목이 다크모드에서 지나치게 밝게(흰색) 뜨던 현상을 `bg-indigo-500/10`의 은은한 틴트 컬러로 교체하여 시각적 피로도 감소.

### 3. 헤더 및 사이드바 프리미엄 디자인 리팩토링
- **헤더 타이틀 Gradient**: 단색 텍스트를 인디고-퍼플 그라데이션(`bg-clip-text text-transparent`)으로 변경하여 화려한 인상 부여.
- **컨텍스트 아이콘 동기화**: 헤더 타이틀 좌측에 현재 위치한 탭의 아이콘을 실시간 매핑하여 시각적 인지 속도 향상.
- **Active 메뉴 포인터**: 사이드바 활성 메뉴 좌측에 뚜렷한 **하이라이트 바(`w-1.5 border-r`)**를 배치하여 포커스 명확화.
- **조작 텐션(Tension)**: 
  - `Hover` 시 메뉴가 살짝 우측으로 미끄러지는(`translate-x-1`) 인터랙션.
  - `Click` 시 전체 버튼이 물리적으로 눌리는 듯한(`active:scale-[0.97]`) 반동 효과.
  - `Active` 탭 아이콘 1.1배 팽창(`scale-110`) 시각 효과.
