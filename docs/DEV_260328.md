## Phase 4 — Settings 페이지 리디자인

### 문제

- 외부 컨테이너 `bg-white/60 border-t` → 박스 안 박스 중첩으로 어색
- 탭바 `bg-white/80` 추가 레이어로 3중 배경
- underline 탭 스타일이 다른 페이지 톤과 불일치

### 해결

1. **외부 컨테이너 제거**: 배경/테두리 없이 앱 배경 그대로 노출
2. **탭 → Pill 컨테이너**: `bg-black/5 dark:bg-white/8 rounded-2xl p-1` 안에 `rounded-xl` 버튼. active: `bg-white dark:bg-white/15 shadow-sm`
3. **섹션 내용 → Glass Card**: Profile / API Keys 내용을 각각 `bg-white/60 dark:bg-white/5 rounded-2xl` card로 감싸 다른 페이지 카드와 일관성
4. **Pipeline**: `-m-8` 해킹 제거 → `flex-1 min-h-0` 구조
5. **Pipeline 배경 dot 패턴 제거**: `<Background variant={Dots}>` 컴포넌트 삭제, 라이트/다크 동일 처리

## Phase 5 — 헤더 Notification 시스템

### 기획 배경

AutoEval은 PDF 임베딩, 계층 태깅, QA 생성·평가 등 장시간 비동기 작업이 많아
작업 완료 시점을 사용자에게 명확히 알려줄 필요가 있음.
Settings의 Notifications 탭은 제거하고 헤더 Bell 아이콘으로 통합.

### 알림 발생 이벤트

| 이벤트 | 메시지 예시 | 타입 |
|---|---|---|
| PDF 임베딩 완료 (`onUploadComplete`) | 임베딩 완료 · `doc_v3.pdf` | success |
| 계층 태깅 완료 (`onTaggingComplete`) | 계층 태깅 완료 | success |
| QA 생성 · 평가 시작 (`onEvalComplete`) | QA 생성 완료 · 평가 시작됨 | success |
| 에러 발생 (향후) | 생성 실패 · API 오류 | error |

### 메시지 포맷

```
[타입 아이콘]  제목 (font-medium)          상대 시간
              부제목 (text-xs, 옵션)
```

상대 시간 표기: `방금 전` / `N분 전` / `N시간 전` / `N일 전`
절대 시간은 title 속성(hover)으로 노출.

### 데이터 구조

```ts
interface Notification {
  id: string;          // crypto.randomUUID()
  title: string;
  sub?: string;        // 파일명, 부가 정보
  type: 'success' | 'error' | 'info';
  time: Date;
  read: boolean;
}
```

### 상태 관리 (in-memory, Option A)

```
App.tsx
  notifications: Notification[]
  addNotification(n)  — 신규 알림 prepend
  clearAll()          — 전체 삭제
  markAllRead()       — 드롭다운 열 때 일괄 read = true

  ↓ props
Header.tsx
  Bell 아이콘 + unread 뱃지 (indigo dot)
  클릭 → 드롭다운 (w-80, max-h-72 scroll)
  "전체 삭제" 버튼 (알림 있을 때만)
  빈 상태: Bell 아이콘 + "알림이 없습니다"
```

### 타입별 스타일

| 타입 | 아이콘 | 색상 |
|---|---|---|
| success | `CheckCircle2` | `text-emerald-500 dark:text-emerald-400` |
| error | `XCircle` | `text-red-500 dark:text-red-400` |
| info | `Info` | `text-indigo-500 dark:text-indigo-400` |

### 변경 파일

| 파일 | 작업 |
|---|---|
| `frontend/src/components/layout/Header.tsx` | Bell 아이콘, 드롭다운, `Notification` 타입 export |
| `frontend/src/App.tsx` | notifications 상태, addNotification/clearAll/markAllRead, 콜백 연결 |
| `frontend/src/components/settings/SettingsPanel.tsx` | Notifications 탭 제거 (Bell import 포함) |

### 헤더 버튼 UI 시각적 위계 정리 (260329 연장 작업)

알림(메인)과 다크모드 선택(보조) 간의 기능적 위계를 명확히 하기 위해 헤더 우측 바 UI 시각적 개선 (Option D):
- **버튼 간격 확대 (`gap-2`)**: 오클릭 방지 및 여백 확보
- **알림 기능 강조**: 아이콘 크기(`w-5 h-5`) 및 색상 대비(`slate-600`)를 높여 주 기능임을 명시
- **시각적 분리**: 두 버튼 사이에 흐린 세로 구분선(`|`) 추가하여 성격 분리
- **보조 버튼 위계 하락**: 다크모드 토글 버튼은 기본 투명도 `opacity-80` 및 연한 색상 처리로 시각적 부담 축소 (hover 시 선명해짐)
