# QA Pipeline 카테고리 드롭다운 표시 문제 수정 — 2026-05-22

QA Pipeline 페이지의 "카테고리 범위 선택" 드롭다운을 펼쳤을 때 하단 옵션이 가려지거나, 2번 "QA 생성" 카드와 겹친 영역에서 옵션 선택이 되지 않는 문제를 수정했다.

---

## 증상

- H1/H2/H3 카테고리 드롭다운 옵션이 많을 때 하단부가 카드 밖에서 잘려 보임.
- `Step 1: 생성 설정`의 드롭다운이 `Step 2: QA 생성` 영역 위로 시각적으로 겹쳐 보이지만, 겹친 부분의 옵션 클릭이 동작하지 않음.

---

## 원인

| 위치 | 원인 |
|------|------|
| `StepCard` 컨테이너 | `overflow-hidden`으로 인해 absolute 드롭다운이 카드 경계 밖에서 잘림 |
| `CustomSelect` 메뉴 | 최대 높이와 내부 스크롤이 없어 옵션이 많으면 화면/카드 하단으로 빠짐 |
| 단계 카드 stacking order | 뒤에 렌더링되는 `Step 2` 카드가 `Step 1` 드롭다운보다 포인터 레이어상 위에 있어 클릭을 가로챔 |

---

## 수정 파일

`frontend/src/components/generation/QAGenerationPanel.tsx`

---

## 변경 내용

### 1. 드롭다운 메뉴 내부 스크롤 추가

`CustomSelect`의 옵션 메뉴에 최대 높이와 세로 스크롤을 추가했다.

```tsx
max-h-72 overflow-y-auto overscroll-contain
```

효과:

- 옵션이 많아도 메뉴가 과도하게 길어지지 않음.
- 하단 옵션은 메뉴 내부 스크롤로 접근 가능.

### 2. StepCard overflow 완화

카드 본문 컨테이너의 `overflow-hidden`을 `overflow-visible`로 변경했다.

효과:

- 드롭다운처럼 카드 바깥으로 떠야 하는 UI가 잘리지 않음.

부수 조정:

- `overflow-hidden` 제거로 카드 헤더의 상단 라운딩이 사라지지 않도록 헤더에 `rounded-t-2xl`을 추가했다.

### 3. 단계별 z-index 지정

`StepCard` 루트에 `relative`와 단계 기반 `zIndex`를 추가했다.

```tsx
<div className="relative flex gap-0" style={{ zIndex: 10 - step }}>
```

효과:

- `Step 1`이 `Step 2`보다 높은 stacking layer를 가진다.
- `Step 1` 드롭다운이 `Step 2` 영역과 겹쳐도 클릭 이벤트가 정상적으로 드롭다운 옵션에 전달된다.

---

## 검증

- 첫 번째 수정 후 `frontend`에서 `npm run lint` 실행 결과 `tsc --noEmit` 통과.
- z-index 추가 후 재실행한 lint는 출력 없이 장시간 진행되어 중단했으나, 변경은 JSX 속성 및 Tailwind class 조정 범위다.
