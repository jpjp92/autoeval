## ✅ 완료된 작업

### 1. RAG 인제스션 운영급 구조 개편 (Phase 2.4) 💎

- **Section-First Architecture**: 페이지 단위 처리를 지양하고, 문서 전체의 **논리적 섹션**을 기준으로 데이터를 그룹화 및 청킹하도록 전면 개편.
- **Robust Heading Detection**: 목차 노이즈(`...`, `···`) 필터링 기능이 강화된 정교한 제목 감지 로직 구현. (기호 비율 30% 이상 자동 제외)
- **Global Section Builder**: 제목을 중심으로 하위 블록들을 계층적으로 묶어주는 트리 빌더 도입.
- **Structured Context Prefix**: 검색 품질 극대화를 위해 모든 청크 상단에 `문서: {명칭} \n 섹션: {경로} \n 페이지: {번호}` 포맷 주입.
- **Metadata 가용성 강화**: `section_title`, `section_path`, `keywords` 등 풍부한 메타데이터 필드 도입 및 중복 데이터 해시 처리.

### 2. 긴급 패치 (Hotfixes) 🔥

- **NameError (page_num)**: PDF 텍스트 추출 시 페이지 번호 변수 미정의 오류 수정.
- **TOC Noise Leakage**: 제목 감지 로직에서 목차의 점선이 포함된 라인을 더 엄격하게 걸러내도록 정규표현식 및 심볼 비율 필터 강화.
- **Utility Function Restore**: 실수로 삭제되었던 `normalize_for_hash` 함수 복구 및 중복 정의된 `is_toc_chunk` 정리.

### 3. TOC 3-단계 필터링 완성 (Phase 2.5) 🔴

기존 `is_toc_chunk()` 함수가 정의만 되어 있고 실제 파이프라인에서 **전혀 호출되지 않던 구조적 문제** 해결.
파이프라인 전 단계에 목차 필터를 삽입하여 목차 청크가 임베딩되는 문제를 완전 차단.

**변경 위치 및 내용**

| 단계                    | 함수                       | 변경 내용                                                                                            |
| ----------------------- | -------------------------- | ---------------------------------------------------------------------------------------------------- |
| **Page Level**    | `extract_text_by_page()` | 앞 1~2페이지가 TOC로 판단되면 해당 페이지 블록 전체 스킵                                             |
| **Heading Level** | `detect_heading()`       | `is_toc_chunk()` 선행 차단 추가 — 목차 항목이 heading으로 인식되어 잘못된 섹션 생성되는 문제 해결 |
| **Section Level** | `process_and_ingest()`   | 섹션 텍스트 전체가 TOC로 판단되면 해당 섹션 통째로 스킵                                              |
| **Chunk Level**   | `process_and_ingest()`   | 청크 단위 `is_toc_chunk()` 필터 추가 **(기존 누락된 핵심 수정)**                             |

**`is_toc_chunk()` 감지 로직 강화 (3-Layer)**

- Layer 1: 심볼 비율 > 0.4 (기존 유지)
- Layer 2: `\.{5,}` 및 `·{5,}` 점선 패턴 직접 감지 추가
- Layer 3: 라인 끝 페이지 번호 패턴 `\s\d{1,3}$` 추가 — `"제1장 작성 배경 07"` 형태 목차 항목 감지

### 4. Header/Footer Bleed-through 3-Layer 필터 (Phase 2.6) 🔴

PyMuPDF가 페이지 header/footer를 일반 텍스트 블록으로 반환하는 특성으로 인해 **본문 청크 끝에 다음 페이지 헤더(`제2권 AI 데이터 구축 가이드 v3.5`)가 포함되는 문제** 해결.
이 노이즈가 embedding에 포함되면 vector similarity가 왜곡되어 관련 없는 query에도 해당 청크가 검색되는 치명적 문제가 발생함.

**신규 함수**

| 함수                                            | 역할                                                                               |
| ----------------------------------------------- | ---------------------------------------------------------------------------------- |
| `detect_repeated_headers(pages, threshold=3)` | 문서 전체 페이지에서 3회 이상 반복 등장하는 줄을 header/footer로 탐지,`set` 반환 |
| `remove_footer_noise(text, repeated_headers)` | 단독 페이지 번호(`"7"`, `"07"`) 및 반복 header 텍스트를 청크에서 제거          |

**3-Layer 적용 위치**

| Layer             | 위치                                            | 방법                                                                                                   |
| ----------------- | ----------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **Layer 1** | `extract_text_by_page()` — 블록 수집 루프 내 | bbox 기반 필터: 상단 8%(`y0 < page_height * 0.08`) 및 하단 8%(`y1 > page_height * 0.92`) 블록 스킵 |
| **Layer 2** | `process_and_ingest()` — 파이프라인 진입부   | `detect_repeated_headers(pages)`로 반복 header 집합 사전 탐지                                        |
| **Layer 3** | `process_and_ingest()` — 청크 생성 후        | `remove_footer_noise(chunk_text, repeated_headers)`로 잔존 노이즈 제거, 이후 50자 품질 필터 적용     |

| 2 | `extract_text_by_page()` 개선 | 테이블 감지 → 분리 처리 통합 | 기존 함수 수정 |

**사용 API**: `page.find_tables()` + `table.extract()` (PyMuPDF 내장)
