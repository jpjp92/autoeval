## 3. Phase 2 잔존 항목 현황

### 3.1 `generators/job_manager.py` — 상세 검토 및 부분 수정

> `ingestion/job_manager.py`는 별도 구현됨 (threading.Lock 적용, IngestionStatus enum 포함). 이슈 없음.
> 아래는 `generators/job_manager.py` 한정.

**코드 검토 결과:**

| # | 항목 | 위치 | 실제 위험도 | 상태 |
|---|------|------|:-----------:|:----:|
| 3 | `jobs` dict TTL/eviction 없음 — 메모리 누수 | `generators/job_manager.py` | 중 (장기 운영) | 미착수 |
| 4 | `get_job()` lock 미적용 — 객체 참조 반환 후 타 스레드 write 교차 가능 | `generators/job_manager.py` L60 | 낮 | ✅ 수정 완료 |
| 5 | `to_dict()` enum 직렬화 — `JobStatus(str, Enum)` 상속으로 실제 문제 없음 | `generators/job_manager.py` | 없음 | 이슈 아님 |
| - | `completed_at` 초기값 `""` — `Optional[str]` 타입과 불일치 | `generators/job_manager.py` L34 | 낮 | ✅ 수정 완료 |

**수정 내용 (2026-04-15):**

```python
# Before
def get_job(self, job_id: str) -> Optional[GenerationJob]:
    return self.jobs.get(job_id)  # lock 없음

completed_at: Optional[str] = ""  # None이어야 일관성 있음

# After
def get_job(self, job_id: str) -> Optional[GenerationJob]:
    with self.lock:
        return self.jobs.get(job_id)

completed_at: Optional[str] = None
```

**`to_dict()` 재확인:** `JobStatus(str, Enum)` — str 상속으로 `asdict()` 직렬화 시 이미 문자열 반환. 변경 불필요.
