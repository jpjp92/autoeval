## 3. 429 예외처리 일관화 (DEV_260413 §6 전체 구현)

### 3.1 설계 방침 재확인

- **429 / 비용 소진**: 즉시 `APIQuotaExceededError` raise + `logger.error`. 재시도·fallback 없음.
- **일시적 오류** (네트워크, JSON 파싱 등): 기존 처리 유지.
- **Silent fallback 제거**: 평가 모듈의 기본 점수(0.65/0.5) 반환 — 429에 한해 raise로 대체.

### 3.2 구현 대상 파일

| 순서 | 파일 | 변경 내용 | 상태 |
|:---:|------|---------|:----:|
| 1 | `backend/exceptions.py` (신규) | `APIQuotaExceededError`, `APIAuthError` 공통 정의 | ✅ |
| 2 | `backend/generators/qa_generator.py` | class 정의 제거 → `exceptions.py` import | ✅ |
| 3 | `backend/generators/worker.py` | import 교체, fallback 로직 제거 (`QUOTA_FALLBACK_MODELS`, `_get_fallback_model` 제거) | ✅ |
| 4 | `backend/ingestion/pipeline.py` | 임베딩 retry 루프에 429 감지 분기 추가 | ✅ |
| 5 | `backend/ingestion/llm_chunker.py` | `_process_batch` `except Exception` 앞 429 감지 추가 | ✅ |
| 6 | `backend/evaluators/rag_triad.py` | 4개 evaluate 함수 `except` 앞 429 감지 추가 | ✅ |
| 7 | `backend/evaluators/qa_quality.py` | `_call_llm_combined`, `evaluate_all` 429 감지 추가 | ✅ |
