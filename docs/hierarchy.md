---

## 🖥️ 대시보드 (index.html)

### 개요

리액트 기반의 인터랙티브 평가 대시보드로, 카테고리별 QA 평가 결과를 시각화하고 분석할 수 있습니다.

### 주요 기능

#### 1. **카테고리 탐색 (좌측 사이드바)**

- 계층적 트리 구조로 모든 카테고리 표시
- 검색 기능으로 빠른 카테고리 조회
- 폴더/태그 아이콘으로 구조 시각화
- 접기/펼치기 기능으로 UI 최적화

#### 2. **대시보드 분석**

- **정성적 평가 (Qualitative)**
  - 정합성/근거성 (30점 만점)
  - 정확성 (25점)
  - Q&A 정합성 (20점)
  - 완전성 (15점)
  - 명료성 (5점)
  - 인용 적절성 (5점)
- **검색 품질 평가 (Retrieval)**
  - 컨텍스트 재현율 (Recall)
  - 컨텍스트 관련성 (Relevance)
  - 컨텍스트 정밀도 (Precision)

#### 3. **테스트 케이스 상세 분석**

- 전체/성공/실패 필터링
- 각 케이스의 점수, 평가 사유, 근거 청크(chunk) 확인
- 타임스탐프 기반 기록

## 🏗️ 전체 카테고리 체계

전체 5개의 대분류로 구성되며, 각 대분류는 세부 카테고리로 계층적으로 확장됩니다.

### 1️⃣ **Shop** - 상품 구매 및 개통

직접 상품을 구매하거나 서비스를 신청할 수 있는 영역

#### 주요 서브 카테고리

- **USIM/eSIM 가입**: 선불 USIM 구매, 휴대폰 요금제 가입 (USIM, eSIM), 이동성 서비스
- **모바일 가입**: 스마트폰 (iPhone, Galaxy 등), 태블릿, 스마트워치, 노트북 에그, 액세서리
- **부가 서비스**: 데이터쉐어링, 듀얼번호, 스마트기기 요금제
- **마이샵 이용안내**: 인터넷TV 가입, 휴대폰 구매 안내
- **요고 다이렉트**: USIM 가입/구매, eSIM 가입, 핸드폰 등록 및 요금제 변경
- **핫딜기획전**: 특가 상품, K-TMI 웹진

**카테고리 수**: 약 30개

## 🗂️ 계층 구조

### 계층 깊이별 분포

**1단계 (대분류)**

- Shop, 고객지원, 상품, 혜택, 마이 (5개)

**2단계 (중분류)**

- 각 대분류별 주요 서브 카테고리
  - 예) Shop > USIMeSIM 가입, Shop > 모바일 가입
  - 예) 상품 > AI, 상품 > TV, 상품 > 로밍

**3단계 (소분류)**

- 더 세분화된 서비스 영역
  - 예) 고객지원 > 공지이용안내 > 서비스안내 > 상품가입
  - 예) 상품 > 모바일 > 요금제 > 5G

**4단계 이상 (세부항목)**

- 구체적인 상품이나 기능
  - 예) 상품 > 모바일 > 요금제 > 5G > 전체
  - 예) 고객지원 > 공지이용안내 > 서비스안내 > 상품가입 > KT모바일전화번호관리기준 > KT모바일010MappingTable보기

### 데이터 품질 평가

#### ✅ 장점

- **최신 정보**: 실제 운영 중인 웹페이지의 스냅샷
- **완전성**: 완전한 페이지 구조 포함
- **구체성**: 상세한 제품/서비스 정보
- **사용자 경험 반영**: 실제 사용자가 보는 콘텐츠

#### ❌ 문제점

1. **비정형 데이터**

   - HTML 마크업과 마크다운의 혼재
   - 일관성 없는 포맷팅
   - JavaScript 함수 호출이 그대로 포함

   ```markdown
   [링고](javascript:kt_common.ktMenuLinkStat('/aiCallLingo.do','^www^KT-개인^상품^AI^AI통화서비스^링고^','_self','BEPE00'); "선택됨")
   ```
2. **정규화 필요**

   - 이미지 절대경로 vs 상대경로 불일치
   - 공백 및 줄바꿈 불규칙
   - 중복된 콘텐츠 제거 필요

   ```
   Line wrapping으로 인한 불완전한 URL/텍스트
   ```
3. **범위 제한**

   - 개별 페이지만 크롤링 (전체 사이트 맵 없음)
   - 동적 로딩 콘텐츠 미포함 가능성
   - 페이지 로드 시점의 스냅샷만 반영
4. **유지보수 문제**

   - 웹사이트 업데이트 시 자동 반영 안 됨
   - 대용량 파일로 검색/처리 성능 저하
   - 버전 관리 어려움
