## 6. 429 예외처리 개선 계획

### 6.1 현황 및 문제점

| 모듈 | 현재 처리 | 문제 |
|------|----------|------|
| `generators/qa_generator.py` | `APIQuotaExceededError` raise ✅ | — |
| `generators/worker.py` | 429 → fallback 모델 전환 후 재시도 | 비용 부족 상황에서 다른 모델도 같은 계정이면 동일 문제 발생 가능, 사용자 인지 지연 |
| `ingestion/pipeline.py` | `except Exception` 3회 재시도 | 429 구분 없이 1s→2s 간격 재시도 — 비용 부족엔 의미 없음 |
| `ingestion/llm_chunker.py` | `except Exception` → raw text fallback | 429 조용히 무시, 청킹 품질 저하 인지 불가 |
| `evaluators/rag_triad.py` | `except Exception` → 기본 점수(0.65) 반환 | 429로 평가 미실행돼도 정상 점수처럼 저장 → 결과 왜곡 |
| `evaluators/qa_quality.py` | `except Exception` → 기본값 반환 | 동일 |

### 6.2 설계 방침

**429 (비용 부족 / Rate Limit)**: 즉시 예외 raise + `logger.error` alert. 재시도·fallback 없음.

**근거**:
- 비용 소진은 재시도로 해결 불가 — 즉시 사용자에게 알려야 함
- Silent fallback(기본 점수, raw 청크)은 결과 신뢰성을 훼손하면서 오류를 숨김
- 모든 모듈에서 일관된 동작 필요

**그 외 일시적 오류** (네트워크 타임아웃, JSON 파싱 실패 등): 기존 처리 유지.

### 6.3 모듈별 변경 방향

#### 공통: `backend/exceptions.py` 신규

`APIQuotaExceededError`가 현재 `generators/qa_generator.py` 내부에 정의됨 → 모든 모듈에서 공유할 수 있도록 공통 파일로 이동.

```python
# backend/exceptions.py
class APIQuotaExceededError(Exception):
    """API 비용 소진 또는 Rate Limit 초과 (즉시 중단, 재시도 불가)"""
    pass

class APIAuthError(Exception):
    """API 키 인증 실패 (즉시 중단)"""
    pass
```

#### `generators/worker.py`

- 현재: `APIQuotaExceededError` → fallback 모델 전환 후 재시도
- 변경: fallback 전환 로직 제거. 즉시 `logger.error` + job FAILED 처리

```python
# 변경 전
if isinstance(e, APIQuotaExceededError):
    fallback = _get_fallback_model(current_model)
    if fallback:
        ... # fallback 모델로 재시도

# 변경 후
if isinstance(e, APIQuotaExceededError):
    logger.error(f"[{job_id}] API 비용 소진 (429) — 즉시 중단: {e}")
    if not fatal_error:
        fatal_error.append(str(e))
    return idx, None, str(e)
```

#### `ingestion/pipeline.py` (`_embed_and_save`)

- 현재: `except Exception` 3회 재시도 (429 포함)
- 변경: 429 감지 시 재시도 없이 즉시 raise

```python
for attempt in range(3):
    try:
        res = await gemini_client.aio.models.embed_content(...)
        break
    except Exception as e:
        err_str = str(e)
        if "429" in err_str or "quota" in err_str.lower() or "resource_exhausted" in err_str.lower():
            logger.error(f"[{filename}] 임베딩 API 비용 소진 (429) — 즉시 중단: {e}")
            raise APIQuotaExceededError(f"Gemini Embedding API 한도 초과: {e}") from e
        if attempt == 2:
            raise
        await asyncio.sleep(2 ** attempt)
```

#### `ingestion/llm_chunker.py` (`_process_batch`)

- 현재: `except Exception` → raw text fallback (청킹 품질 저하, 오류 무시)
- 변경: 429 감지 시 즉시 raise. JSON 파싱 실패 등 다른 오류는 기존 fallback 유지.

```python
except Exception as e:
    err_str = str(e)
    if "429" in err_str or "quota" in err_str.lower() or "resource_exhausted" in err_str.lower():
        logger.error(f"Batch {batch_num}: LLM 청킹 API 비용 소진 (429) — 즉시 중단: {e}")
        raise APIQuotaExceededError(f"Gemini LLM Chunking API 한도 초과: {e}") from e
    # 기존 fallback 유지 (JSON 파싱 실패 등)
    logger.error(f"Batch {batch_num}: error ({elapsed:.1f}s): {e}")
    return batch_num, [{"text": b["text"], ...} for b in new_blocks]
```

#### `evaluators/rag_triad.py`, `evaluators/qa_quality.py`

- 현재: `except Exception` → 기본 점수(0.65) 반환
- 변경: 각 평가 함수에서 429 감지 시 즉시 raise. 상위 호출부에서 job FAILED 처리.

```python
except Exception as e:
    err_str = str(e)
    if "429" in err_str or "quota" in err_str.lower() or "resource_exhausted" in err_str.lower():
        logger.error(f"평가 API 비용 소진 (429) — 즉시 중단: {e}")
        raise APIQuotaExceededError(f"평가 모델 API 한도 초과: {e}") from e
    logger.warning(f"Evaluation error (non-quota): {e}")
    return 0.65  # 일시적 오류만 기본값 유지
```
