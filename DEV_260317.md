### [Ingestion] `normalize_text` 강화

```python
# 1. Ÿ (불릿 폰트 아티팩트) 정규화
text = re.sub(r'[ŸŸ]', '-', text)  # 유니코드 변형 모두 처리

# 2. 스마트 줄바꿈 결합 (문장 중간 줄바꿈 → 공백)
# 조건: 이전 줄이 .!?。로 끝나지 않고, 다음 줄이 불릿/표/제목이 아닌 경우
def smart_join_lines(text: str) -> str:
    result, buffer = [], ""
    for line in text.split('\n'):
        stripped = line.strip()
        if not buffer:
            buffer = stripped
        elif (buffer
              and not re.search(r'[.!?。]$', buffer)
              and not re.match(r'^[-•|\[#\d]', stripped)):
            buffer += ' ' + stripped
        else:
            result.append(buffer)
            buffer = stripped
    if buffer:
        result.append(buffer)
    return '\n'.join(result)
```

### [Ingestion] 짧은 청크 병합

스플리터 후 후처리 단계에서, 같은 `section_path` 내 인접 청크가 200자 미만이면 다음 청크와 병합 (최대 `chunk_size`까지).

```python
def merge_short_chunks(chunks: list, min_chars: int = 200, max_chars: int = 1200) -> list:
    merged, buf = [], ""
    for chunk in chunks:
        if not buf:
            buf = chunk
        elif len(buf) + len(chunk) < max_chars and len(buf) < min_chars:
            buf += "\n" + chunk
        else:
            merged.append(buf)
            buf = chunk
    if buf:
        merged.append(buf)
    return merged
```

CREATE OR REPLACE VIEW qa_pairs_view AS
SELECT
  qg.job_id, qg.created_at,
  doc->>'docId'                AS doc_id,
  doc->'hierarchy'             AS hierarchy,
  qa->>'q'                     AS question,
  qa->>'a'                     AS answer,
  qa->>'intent'                AS intent,
  (qa->>'answerable')::boolean AS answerable

  jsonb_array_elements(qg.qa_list) AS doc,

### 현재 데이터 흐름 (As-Is)

```
[경로 1: JSON Fallback] — 현재 실제 동작 경로
  ref/data/data_2026-03-06_normalized.json (1,106개 KT 웹 스크래핑 문서)
    └─ { docId, hierarchy: ["Shop","USIM","선불USIM"], title, text(웹본문), url }
         └─ generate_qa(item)
              ├─ text[:2000] → 프롬프트에 삽입
              └─ hierarchy 문자열 조합 → [카테고리] 라벨

[경로 2: Vector DB] — hierarchy 필터가 있을 때만 진입
  Supabase doc_chunks
    └─ search_doc_chunks(dummy_zero_vector, filter={l1/l2/l3})
         └─ { id, content, metadata.hierarchy_l1/l2/l3 }
              └─ item = { docId: id, hierarchy: [l1,l2,l3], text: content }
```

### 문제점 분류

#### 🔴 크리티컬 (QA 생성 품질 치명적)

**1. 프롬프트 도메인 하드코딩**
```python
# config/prompts.py 현재
SYSTEM_PROMPT_KO_V1 = """당신은 통신사 고객지원 QA 데이터셋 생성 전문가입니다."""
```
→ 현재 문서(AI 데이터 구축 가이드)와 전혀 맞지 않음. **범용 문서 도메인**으로 교체 필요.

**2. DB 청크 text 필드에 context prefix 포함**
```python
# backend/main.py line 278
text = item.get("text", "")[:2000]
```
DB 청크의 `content`에는 `[문서] ... [섹션] ... [페이지] ...\n\n[내용]\n...` 형식의 prefix가 포함됨.
`text[:2000]`으로 그대로 프롬프트에 넣으면 LLM이 구조 메타 텍스트를 본문으로 오인.
→ `raw_text` (순수 본문)를 별도 저장하거나, prefix를 제거하는 전처리 필요.

**3. dummy zero vector — semantic 검색 무력화**
```python
# generation_api.py line 303
if query_vector is None:
    query_vector = [0.0] * 1536  # Dummy for metadata filter only
```
→ `match_doc_chunks` RPC가 내부적으로 cosine similarity를 계산하므로, zero vector는 결과가 불확정적.  
→ Supabase `.table("doc_chunks").select().eq("metadata->>hierarchy_l1", h1)` 방식의 직접 필터로 교체하거나,  
  L1/L2 레이블 텍스트 자체를 embedding해서 semantic query로 활용.

#### 🟡 중요 (DB 기반 워크플로우 미완성)

**4. 프론트엔드 hierarchy 하드코딩**
```tsx
// QAGenerationPanel.tsx — 현재
const hierarchy = ["Shop", "USIM/eSIM 가입", "선불 USIM 구매/충전"];
```
→ DB에 있는 실제 L1/L2 목록을 동적으로 가져오는 API 및 선택 UI 없음.  
→ `generateQA()` 호출 시 `hierarchy_l1/l2/l3` 파라미터도 미전달 (항상 JSON fallback 경로).

**5. JSON fallback 의존성**
→ `ref/data/data_2026-03-06_normalized.json`은 KT 통신사 데이터. 범용 시스템으로 전환되면 무의미.  
→ fallback을 명시적 오류 또는 deprecation 경고로 교체해야 DB 경로만 동작하게 됨.

#### 🟢 개선 (선택적)

**6. chunk_type별 QA 전략 미분화**
- `table` 청크 → 수치 비교형 QA (`numeric`, `list` 의도)
- `list` 청크 → 절차형 QA (`procedure`, `how` 의도)
- `body` 청크 → 설명형 QA (`factoid`, `why`, `definition` 의도)
- `heading` 청크 단독 → QA 생성 의미 없음 (skip)

현재는 모든 청크에 동일한 8-intent 프롬프트 적용.

### JSON(기존) vs DB(신규) 스키마 비교

| 항목 | JSON 기존 | DB chunk 신규 |
|------|-----------|----------------|
| 텍스트 필드 | `item["text"]` (웹 원문 수KB) | `content` (청킹 200~1200자) |
| context prefix | 없음 | `[문서]...[섹션]...[페이지]...` 포함 |
| hierarchy | `["Shop","USIM","선불USIM"]` (URL 기반) | `hierarchy_l1/l2/l3` (AI 태깅) |
| 문서 식별자 | `docId: "ktcom_3842"` | `id: UUID` |
| 추가 메타 | `url`, `title`, `startdate`, `status` | `chunk_type`, `section_path`, `keywords`, `page` |
