### 2. 이미지 기반 PDF 업로드 시 0 chunks 문제

**증상**: `테스트데이터_4.pdf` 업로드 후 `Enhanced Ingestion Complete: 0 chunks` → hierarchy 분석 404 오류

**근본 원인**: PDF가 `\uf09f`(Private Use Area 문자) 기반 심볼 폰트 사용 → PyMuPDF가 의미 있는 텍스트를 추출 못함

- `build_sections`: 2 sections만 생성
- `Section[0] 'Root'`: 21 blocks, text_len=41 (블록당 평균 2자)
- `Section[1] '\uf09f ․ ․'`: 64 blocks, text_len=471 → 전부 symbol noise 필터

**디버그 로깅 추가** (`backend/api/ingestion_api.py`):

- `build_sections` 결과 섹션 수 로깅
- 섹션[0~4] heading/blocks/text_len 샘플 로깅
- 필터별 카운터: `{'toc': N, 'colophon': N, 'symbol': N, 'too_short': N, 'duplicate': N}`

**수정** (`backend/api/ingestion_api.py`):

```python
# 업로드 직후 텍스트 밀도 사전 검사
total_text = "".join(b.get("text","") for p in pages for b in p.get("blocks",[]))
avg_chars_per_block = len(total_text) / max(total_blocks, 1)
if len(total_text) < 300 or avg_chars_per_block < 5:
    raise HTTPException(400, "PDF에서 텍스트를 추출할 수 없습니다. 이미지 기반 PDF이거나 커스텀 심볼 폰트를 사용하는 문서입니다.")
```

- 0 chunks 완료 시 `logger.error` + 조기 return (기존 `logger.info` → 개선)

**원인 2**: 기존 `qa_gen_results.metadata`에 `source_doc` 미저장 → 히스토리 조회 시 빈값

**수정 내용**:

| 파일                                                             | 수정 내용                                                                                                                                                    |
| ---------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `frontend/src/lib/exportUtils.ts`                              | 모든 `?? model ?? '-'` → `\|\| model \|\| '-'` (3곳: CSV/HTML/JSON)                                                                                         |

- `isAnimationActive={false}` → `true` 변경
- recharts 내장 시계방향 sweep 애니메이션 활성화

#### 품질 점수 (커스텀 바 차트)

**핵심 문제**: 컴포넌트가 unmount되지 않음 (App.tsx: `hidden` 클래스로 탭 전환). recharts는 `ResponsiveContainer` 내부 `ResizeObserver`로 크기 변화(hidden→visible: 0px→실제px)를 감지해 재애니메이션하지만, 커스텀 바 차트에는 이 메커니즘이 없었음.

**시도 및 실패 이력**:

| 시도 | 방법 | 실패 원인 |
|------|------|-----------|
| 1차 | `useState(animated)` + `useEffect([data])` | React 렌더 배치로 0%→X% transition 무시 |
| 2차 | `useRef` + 이중 RAF + DOM 직접 조작 | React reconciler가 다음 렌더에 0% 덮어씀 |
| 3차 | `flushSync` + `widths` state | effect cleanup이 timer 취소 → `animated` 영원히 false |
| 4차 | `clip-path` + `flushSync` | effect cleanup + prevKeyRef 조합으로 timer 취소 → 바 완전히 숨김 |

**최종 해결** (`clip-path` + `ResizeObserver`):

```tsx
// 두 가지 트리거를 분리
// 1) data 값 변경 시 (히스토리 전환 등)
useEffect(() => {
  const key = data.map(d => d.score.toFixed(4)).join(',');
  if (key === '' || prevKeyRef.current === key) return;
  prevKeyRef.current = key;
  triggerFnRef.current();
}, [data]); // cleanup 없음 — cleanup이 timer 취소하면 animated 영원히 false

// 2) 탭 전환으로 hidden→visible 될 때 (recharts와 동일 원리)
useEffect(() => {
  const ro = new ResizeObserver(() => {
    const visible = el.offsetWidth > 0;
    if (visible && wasHiddenRef.current) { wasHiddenRef.current = false; triggerFnRef.current(); }
    else if (!visible) { wasHiddenRef.current = true; }
  });
  ro.observe(containerRef.current);
  return () => ro.disconnect();
}, []);
```

**애니메이션 방식**: `clip-path: inset(0 100% 0 0)` → `inset(0 0% 0 0)` (왼→오 채우기)
- `width`는 target값으로 고정, clip-path만 애니메이션
- `i * 100ms` stagger로 바 순차 등장

**최종 동작**:

| 상황 | 도넛 | 방사형 | 바 |
|------|------|--------|-----|
| 다른 탭 → evaluation 진입 | ✅ | ✅ | ✅ |
| 히스토리 항목 전환 | ✅ | ✅ | ✅ |
| 상세 테이블 페이지 이동 | ❌ | ❌ | ❌ |

### 21. L2/L3 과다 생성 원인 분석

**증상**: hierarchy tagging 후 L2·L3가 문서 청크 수만큼 증가하는 케이스 발생

**원인**:

| 원인 | 내용 |
|------|------|
| L2 사전 목록 없음 | L1과 달리 확정 목록 없이 청크마다 자유 생성 |
| "섹션 제목 반복 금지" 지시 | 창의적 L2 표현을 유도 → 배치 간 표현 불일치 |
| 배치 독립 처리 | 5청크씩 독립 처리 → 배치 간 L2 이름 일관성 미보장 |
| 후처리 통합 없음 | tagging 후 유사 L2/L3 병합 단계 없음 |

**개선 방향** (미구현):
- 2-pass 구조: L1 확정 → L2 후보 3~5개 확정 → 청크별 확정 목록에서 선택
- 단기 대안: 프롬프트에 L2 수 제한 + 이전 배치 L2 목록 전달
