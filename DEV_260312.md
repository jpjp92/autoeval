## 🔧 다음 작업: 병렬 평가 처리 (ThreadPoolExecutor)

### 배경
- 현재 Layer 2 (RAG Triad), Layer 3 (Quality) 모두 **QA 단위 순차 처리**
- 16 QA × 3 메트릭 = 48번의 API 호출이 직렬로 실행 → 속도 병목
- `ThreadPoolExecutor`로 QA 단위 병렬화하면 최대 N배 단축 가능

### 적용 대상

| Layer | 현재 | 목표 |
|---|---|---|
| Layer 1-A Syntax | 순차 (거의 즉시) | 유지 (비용 없음) |
| Layer 1-B Stats | 순차 (거의 즉시) | 유지 (비용 없음) |
| **Layer 2 RAG Triad** | 순차 for loop | **병렬화** |
| **Layer 3 Quality** | 순차 for loop | **병렬화** |
